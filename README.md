# Генератор расклеек СПН «Этажи»

Статический инструмент для GitHub Pages, который помогает специалисту по недвижимости пройти путь от рабочей задачи до печати, задания на расклейку и отчёта.

## Основной сценарий

1. Выбрать режим работы.
2. Выбрать задачу, сценарий и формат А4.
3. Выбрать готовый безопасный шаблон.
4. Заполнить данные объекта и контакты СПН.
5. Добавить фото или QR при необходимости.
6. Проверить качество и рекомендации менеджера.
7. Напечатать или сохранить PDF.
8. Сформировать задание на расклейку.
9. Зафиксировать план, факт, точки и проблемы.
10. Передать результат в отчёт и аналитику.

## Офисный сценарий для Борисоглебска

Генератор развивается как рабочий инструмент отдела продаж, а не только как редактор объявлений.

Рекомендуемый путь:

1. Менеджер определяет рекомендованные шаблоны для новичков.
2. СПН выбирает рабочую ситуацию: собственник, покупатель, объект, новостройка, консультация или личный бренд.
3. СПН подставляет контакты и сведения об объекте.
4. Макет проходит контроль качества.
5. Рискованный шаблон или первый макет новичка проверяет менеджер.
6. После печати создаётся задание на расклейку.
7. Исполнитель отмечает точки, количество листов и проблемы.
8. Итог переносится в отчёт и аналитику.

## Что уже умеет генератор

### Шаблоны и офисные сценарии

- шаблоны по задачам, рабочим ситуациям и сценариям;
- 21 локальный сценарий Борисоглебска;
- пакеты по ЖК, подъездной расклейке, доверию и A/B-проверкам;
- поиск по названию, тегам, источнику и видимому тексту;
- office-метаданные: уровень, риск, сценарий, формат А4 и заметка менеджера;
- жизненный цикл `working / test / deprecated`;
- уникальные итоговые `templateId`, `sourceId` и `sourcePackage`;
- безопасные замены устаревших шаблонов;
- запрет неподтверждённых обещаний готового покупателя, гарантированной продажи, цены, ипотеки и результата.

### Интерфейс и пошаговая подготовка

- режимы `Новичок`, `Быстро`, `Расширенно`;
- режим `Новичок` с безопасным фильтром шаблонов;
- в режимах `Новичок` и `Быстро` задача определяется рабочей ситуацией без повторного блока выбора;
- ручной выбор задачи и полный набор фильтров остаются в `Расширенном` режиме;
- маршрут `Задача и формат → Шаблон → Текст → Фото / QR → Проверка и печать → Задание → Отчёт`;
- контекстная кнопка следующего шага;
- заметный keyboard-focus;
- управление карточками и компоновками с клавиатуры;
- доступный печатный диалог;
- блок проверки менеджера перед печатью;
- ручная регрессионная проверка режима «Новичок»;
- адаптивная работа на мобильном экране.

### Макет и печать

- официальные пресеты `1, 2, 3, 4, 6 и 8 макетов на А4`;
- неподдерживаемые 10/12 блокируются при импорте;
- форматы 3 и 8 имеют отдельные компактные профили;
- для 6 и 8 автоматически отключаются отрывные полосы;
- единый browser-контракт проверяет телефон, имя СПН, overflow, контакты и полосы для каждого формата;
- отдельные CDP screenshot-сценарии для плотных форматов 3, 6 и 8;
- линии реза, безопасные поля и экономный цвет;
- настоящий QR без внешнего сервера;
- фото-компоновки `photo_left`, `photo_card`, `newbuild_visual`, `agent_brand_photo`;
- вычисляемая команда `Вписать`, учитывающая реальную рабочую область;
- сводка перед печатью с телефоном, форматом и предупреждениями.

### Сохранение и перенос

- автосохранение текущей работы на этом устройстве;
- отдельный ручной резерв одного текущего макета;
- раздел «Мои макеты» для нескольких именованных вариантов на этом устройстве;
- файл одного макета для передачи без истории и настроек;
- полная копия рабочего пространства для переноса профиля, макетов, настроек, заданий и отчётов;
- версионированный JSON-макет с диагностикой ошибок;
- защита от тихого перезаписывания одинакового названия;
- резервная точка перед очисткой, импортом и загрузкой;
- отмена последнего разрушительного действия;
- подтверждение удаления именованного макета;
- восстановление последнего удалённого макета без перезаписи соседних записей.

### Контроль качества

- проверка телефона, текста, QR, фото, плотности, фирменности и канала отклика;
- метки `ошибка`, `важно`, `совет`;
- прямые безопасные действия для пустого QR, фото и отсутствующего канала отклика: выбранные блоки не выключаются автоматически, а пользователь переходит к нужному полю или включает контакты только при корректном телефоне;
- подавление дублирующих QR-замечаний;
- мягкая автоподстройка, которая сохраняет включённые фото и QR в `auto` и явных режимах подстройки;
- менеджерская office-сводка.

### Задание и полевое выполнение

- задание на расклейку для исполнителя;
- мобильный режим до 30 точек;
- план и факт листов;
- статусы выполнения и проблемы;
- отметка фотоотчёта без хранения изображений;
- перенос полевого итога в отчёт.

### Задание и отчёты

- отчёт, история, фильтр и аналитика;
- история до 30 отчётов;
- сохранение `templateId`, office-контекста и оценки качества;
- CSV-экспорт;
- фильтры `Все / Рабочие / Слабые`;
- аналитика связок `Лучше повторить / Нужно изменить`;
- устойчивые связки `templateId + место`;
- удаление ошибочного отчёта с подтверждением;
- сводка менеджеру за 7 дней, 30 дней или всю историю.

### Полная копия рабочего пространства

- экспорт всех проектных ключей `localStorage`;
- режимы `Объединить` и `Заменить`;
- проверка схемы, версии, размера и ключей;
- откат к предыдущему снимку при ошибке записи;
- полная копия позволяет перенести и восстановить рабочее пространство;
- фото в backup не сохраняются.

## Хранение и приватность

Профиль, макеты, настройки, задания и отчёты хранятся в `localStorage` текущего браузера.

- данные не отправляются на внешний сервер;
- очистка данных браузера удаляет локальное состояние;
- backup-файл может содержать телефоны, адреса и заметки, поэтому его нельзя публиковать;
- в автоматических screenshot используются только синтетические данные.

## Автоматические проверки

### Полный статический набор

```bash
npm run validate
```

Он проверяет:

- шаблоны, уникальные ID и office-метаданные;
- JavaScript, импорты и DOM-id;
- runtime-архитектуру;
- storage, JSON и backup;
- контроль качества и Wizard Flow;
- официальный набор форматов А4;
- release candidate, changelog и ручные блокеры;
- конфигурацию GitHub Actions.

### Browser smoke

```bash
npm run smoke:browser
npm run smoke:ui-actions
```

Проверяются загрузка приложения, режимы интерфейса, Wizard Flow, сохранение и восстановление, клавиатура, печатный диалог и failure artifacts.

### Печатная автоматика

```bash
npm run screenshots:print
node tools/run-eight-print-screenshot.mjs
node tools/run-print-format-coverage-smoke.mjs
```

Основная матрица создаёт семь PNG-сценариев. Формат 8 имеет отдельный PNG. Единый coverage-smoke последовательно проверяет все форматы `1 / 2 / 3 / 4 / 6 / 8`.

Автоматическая проверка не заменяет физическую приёмку issue #40.

# Генератор расклеек СПН «Этажи»

Статический инструмент для GitHub Pages. Цель — чтобы специалист по недвижимости быстро создавал макеты под разные задачи: куплю, продам, услуги, новостройки, аренда, личный бренд, частные объявления.

## Основной сценарий

1. Выбрать режим работы.
2. Выбрать задачу и формат А4.
3. Выбрать готовый безопасный шаблон.
4. Заменить имя, телефон, район, цену и параметры.
5. Добавить фото или QR при необходимости.
6. Проверить качество и рекомендации менеджера.
7. Напечатать или сохранить в PDF.
8. Сформировать задание на расклейку.
9. Зафиксировать результат и сохранить его в историю.
10. Сравнить рабочие и слабые связки.

## Офисный сценарий для Борисоглебска

Генератор развивается как рабочий инструмент отдела продаж, а не просто редактор объявлений.

Рекомендуемый путь внедрения:

1. Менеджер выбирает рекомендованные шаблоны для новичков.
2. СПН начинает с выбора рабочей ситуации: собственник, покупатель, объект, новостройка, консультация или личный бренд.
3. СПН подставляет свои контакты и данные района, дома, ЖК или объекта.
4. Макет проходит контроль качества.
5. Первый макет новичка и рискованный шаблон проверяет менеджер.
6. После печати формируется задание на расклейку.
7. После выполнения заполняется отчёт по откликам.
8. Удачные связки сохраняются и используются повторно.

Подробности:

```text
docs/office-rollout-plan.md                 план внедрения генератора в офисе
docs/spn-one-page-instruction.md            краткая инструкция для СПН
docs/manager.md                             короткий чек-лист менеджера
docs/office-template-filters.md             офисные подборки и теги шаблонов
docs/template-authoring-guide.md            правила шаблонов и office-метаданных
docs/wizard-flow-spec.md                    фактическая логика пошагового мастера
docs/newbie-mode-regression-checklist.md    ручная проверка режима «Новичок»
docs/full-scenario-regression-checklist.md  проверка полного пути СПН
docs/newbie-mode-rollback-plan.md           аварийный откат режима «Новичок»
docs/report-history.md                      история и аналитика расклеек
docs/report-history-regression-checklist.md ручная проверка истории отчётов
```

## Что уже умеет генератор

### Шаблоны и офисные сценарии

- шаблоны по задачам и сценариям;
- стартовые пустые шаблоны для нестандартных макетов;
- избранные шаблоны;
- счётчики шаблонов по сценариям;
- офисные подборки через крупные карточки;
- локальные шаблоны под Борисоглебск;
- отдельный пакет по ЖК «Теллерманов сад»;
- отдельный пакет подъездных шаблонов;
- A/B-тестовые и доверительные шаблоны;
- поиск по названию, тегам и видимому тексту макета;
- office-метаданные для подъездов, Борисоглебска и ЖК: уровень, риск, сценарий, формат А4 и заметка менеджера;
- бейджи `Новичку`, `Проверка`, `Рекомендовано`, риск и формат А4;
- notice для менеджерских и высокорисковых шаблонов.

### Интерфейс и Wizard Flow

- режимы `Новичок`, `Быстро`, `Расширенно`;
- режим `Новичок` с безопасным фильтром шаблонов, пошаговым мастером и защитой печати;
- автоматический запуск пошагового мастера при входе в `Новичок`;
- маршрут `Цель → Заготовка → Текст → Фото / QR → Проверка → Задание → Отчёт` для новичка;
- пропуск скрытого шага сохранения без зацикливания;
- активная подсказка `Что сделать сейчас`;
- контекстная кнопка `Далее: следующий шаг`;
- мягкие рекомендации без блокировки навигации;
- строгая отдельная защита печати новичка;
- блок проверки менеджера перед печатью;
- ручная регрессионная проверка режима «Новичок» по отдельному чек-листу.

### Макет и печать

- пресеты 1, 2, 3, 4, 6 и 8 макетов на А4;
- быстрые режимы с автоматической перестановкой блоков;
- мягкая автоподстройка с сохранением фото и QR;
- управление составом и порядком блоков;
- сохранение профиля СПН, последнего и именованных макетов;
- корректный импорт JSON без смешивания со старым состоянием;
- редактор контактов, подписи отрывных телефонов и брендовой строки;
- настоящий QR без внешнего сервера;
- hero-компоновка фото для режимов `С фото` и `Витрина` на 1–2 макета А4;
- отдельное безопасное поведение планировки;
- линии реза, безопасные поля и проверка печати;
- сводка перед печатью с телефоном, форматом и предупреждениями.

### Контроль качества

- проверка телефона, текста, QR, фото, плотности, фирменности и канала отклика;
- метки `ошибка`, `важно`, `совет`;
- счётчики и фильтры замечаний;
- подсказка первого исправления;
- визуальное состояние кнопки печати;
- безопасные прямые действия без самовольного отключения фото и QR;
- подавление дублирующих QR-замечаний;
- менеджерская office-сводка.

### Задание и отчёты

- задание на расклейку для исполнителя;
- отчёт после расклейки;
- история до 30 отчётов в браузере;
- повтор прошлой расклейки;
- CSV-экспорт;
- автоматический вывод по результату;
- фильтр по месту, макету и заметкам;
- фильтры `Все / Рабочие / Слабые`;
- аналитика связок `Лучше повторить / Нужно изменить`.

## Хранение и приватность

Профиль, макеты, избранное и история отчётов хранятся только в `localStorage` текущего браузера.

- данные не отправляются на внешний сервер;
- фото не сохраняются в localStorage;
- очистка данных браузера удалит локальные макеты и историю;
- для отчётов рекомендуется периодически выгружать CSV;
- полный backup рабочего пространства запланирован отдельным этапом.

## Runtime-архитектура

Постоянные SPN-helper-модули подключаются через:

```text
assets/js/spnUiMode.js
```

Они не должны одновременно подключаться отдельными script-тегами в `index.html`.

`assets/js/templates.js` использует общий Promise-кеш: все модули получают один набор шаблонов и не повторяют загрузку девяти JSON-пакетов.

Правила закреплены проверкой:

```bash
npm run validate:runtime-architecture
```

## Автоматические проверки

Общий запуск:

```bash
npm run validate
```

Он последовательно запускает все `validate:*` из `package.json`, включая:

- шаблоны и office-метаданные;
- синтаксис JavaScript;
- связи файлов, импортов и DOM-id;
- дубли HTML-ассетов;
- runtime-архитектуру;
- storage и импорт/экспорт;
- контроль качества и его действия;
- Wizard Flow и режим `Новичок`;
- историю отчётов и документацию;
- версии и changelog;
- конфигурацию GitHub Actions.

Статические проверки пока не заменяют реальный браузерный smoke-тест. Его добавление — следующий этап master-roadmap.

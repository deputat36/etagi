# Генератор расклеек СПН «Этажи»

Статический инструмент для GitHub Pages. Цель — чтобы специалист по недвижимости быстро создавал макеты под разные задачи: куплю, продам, услуги, новостройки, аренда, личный бренд и частные объявления.

## Основной сценарий

1. Выбрать режим работы.
2. Выбрать задачу и формат А4.
3. Выбрать готовый безопасный шаблон.
4. Заменить имя, телефон, район, цену и параметры.
5. Добавить фото или QR при необходимости.
6. Проверить качество и рекомендации менеджера.
7. Напечатать или сохранить в PDF.
8. Сформировать задание на расклейку.
9. Выполнить точки в мобильном режиме и зафиксировать план/факт.
10. Передать полевой итог в отчёт.
11. Сохранить результат и сравнить рабочие, слабые и устойчивые связки.

## Офисный сценарий для Борисоглебска

Генератор развивается как рабочий инструмент отдела продаж, а не просто редактор объявлений.

Рекомендуемый путь внедрения:

1. Менеджер выбирает рекомендованные шаблоны для новичков.
2. СПН начинает с рабочей ситуации: собственник, покупатель, объект, новостройка, консультация или личный бренд.
3. СПН подставляет контакты и данные района, дома, ЖК или объекта.
4. Макет проходит контроль качества.
5. Первый макет новичка и рискованный шаблон проверяет менеджер.
6. После печати формируется задание на расклейку.
7. Исполнитель отмечает точки, факт листов, фотоотчёт и проблемы.
8. Полевой итог переносится в отчёт, куда добавляются отклики и целевые обращения.
9. Удачные связки сохраняются и используются повторно.

### Backup рабочего пространства

- экспорт всех проектных ключей `localStorage` с префиксом `etagi-raskleyka-`;
- профиль, макеты, избранное, режим интерфейса, Wizard Flow, задания, проверки менеджера, полевой режим и история отчётов;
- режимы `Объединить` и `Заменить`;
- проверка схемы, версии, размера, ключей и значений;
- подтверждение перед изменением данных;
- откат к предыдущему снимку при ошибке записи;
- отсутствие сетевой отправки;
- фото не сохраняются в backup.

## Хранение и приватность

Профиль, макеты, избранное, настройки, полевое состояние и история отчётов хранятся только в `localStorage` текущего браузера.

- данные не отправляются на внешний сервер;
- фото не сохраняются в localStorage;
- очистка данных браузера удалит локальные данные;
- полный backup позволяет перенести и восстановить рабочее пространство;
- backup-файл может содержать телефоны, адреса и заметки, поэтому его нельзя публиковать;
- для дополнительного анализа отчёты можно выгружать в CSV.

## Runtime-архитектура

Постоянные SPN-helper-модули подключаются через:

```text
assets/js/spnUiMode.js
```

Они не должны одновременно подключаться отдельными script-тегами в `index.html`.

`assets/js/templates.js` использует общий Promise-кеш и объединяет шаблоны, portfolio, office-overrides и ID-aliases.

Правила закреплены проверкой:

```bash
npm run validate:runtime-architecture
```

## Автоматические проверки

### Быстрые статические проверки

```bash
npm run validate
```

Команда последовательно запускает все `validate:*` из `package.json`, включая:

- шаблоны, уникальные ID и office-метаданные;
- покрытие 21 локального сценария Борисоглебска;
- синтаксис JavaScript;
- связи файлов, импортов и DOM-id;
- дубли HTML-ассетов;
- runtime-архитектуру;
- storage, импорт/экспорт и backup;
- мобильное выполнение расклейки;
- keyboard/focus выбора компоновки;
- контроль качества и его действия;
- Wizard Flow и режим `Новичок`;
- историю отчётов и документацию;
- версии, release candidate и changelog;
- сверку evidence-пакета чувствительных шаблонов;
- конфигурацию GitHub Actions.

### Настоящий браузерный smoke

```bash
npm run smoke:browser
```

Smoke-тест запускает локальный сервер и системный Chrome/Chromium. Он проверяет:

- появление шаблонов и листа А4;
- ранние и поздние runtime-ошибки;
- переключение трёх режимов;
- автоматический запуск Wizard Flow;
- переход `Проверка → Задание → Отчёт`;
- наличие backup;
- открытие и закрытие мобильного режима исполнителя;
- keyboard/ARIA выбора компоновки;
- `newbuild_visual`;
- `Частное → Фото СПН`;
- отсутствие неожиданной блокировки основных кнопок.

В GitHub Actions browser smoke работает отдельным job после успешного `npm run validate`.

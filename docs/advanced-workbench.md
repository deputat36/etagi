# Рабочий блок после печати

В расширенном режиме три крупные секции объединяются в один раскрываемый блок `Работа после печати`:

- `План расклейки`;
- `Квалификация отклика`;
- `Итог теста расклейки`.

## Зачем это сделано

Раньше все три карточки были одновременно раскрыты и сильно увеличивали длину левой панели. Теперь основная работа с макетом, проверкой и печатью остаётся выше, а инструменты следующего этапа доступны в одном месте.

## Безопасность данных

Модуль не создаёт копии секций и не переписывает их содержимое. Он переносит существующие DOM-узлы внутрь `<details>`, поэтому сохраняются:

- все обработчики кнопок;
- введённые значения;
- журнал тестов;
- копирование плана и заметок;
- локальное хранение самих инструментов.

Хранится только состояние раскрытия блока под проектным ключом:

```text
etagi-raskleyka-advanced-workbench-open-v1
```

Этот ключ входит в общий backup рабочего пространства.

## Поведение по режимам

- `Расширенно` — блок виден и по умолчанию закрыт;
- `Быстро` — блок скрыт вместе со второстепенными инструментами;
- `Новичок` — блок скрыт, чтобы не перегружать пошаговый маршрут.

## Ручная проверка

1. Открыть режим `Расширенно`.
2. Найти блок `Работа после печати` перед сохранением макета.
3. Раскрыть его и убедиться, что внутри находятся три инструмента в правильном порядке.
4. Заполнить заметку отклика и поля итогов теста.
5. Закрыть и снова раскрыть блок — значения не должны исчезнуть.
6. Перезагрузить страницу — состояние раскрытия должно восстановиться.
7. Переключиться в `Быстро` и `Новичок` — пустой контейнер не должен занимать место.
8. Вернуться в `Расширенно` — инструменты снова доступны.

## Автоматический контракт

```bash
npm run validate:advanced-workbench
```

Проверка запрещает клонирование секций, глобальные обработчики кликов и бесконечный MutationObserver.

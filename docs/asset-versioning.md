# Версионирование entry-assets

## Зачем

GitHub Pages и браузер могут некоторое время использовать кеш CSS и JavaScript. Чтобы после выпуска пользователь получал согласованный набор файлов, все CSS и JS, подключённые непосредственно из `index.html`, имеют единый query-токен:

```text
?v=<package.version>
```

## Обновление

После изменения версии в `package.json` выполнить:

```bash
npm run assets:stamp
```

Команда читает `package.json` и обновляет все локальные ссылки `assets/css/*.css` и `assets/js/*.js` в `index.html`.

Затем выполнить:

```bash
npm run validate
npm run smoke:browser
```

## Автоматический контроль

`validate:asset-version` проверяет:

- каждый CSS/JS entry-asset имеет `?v=`;
- токен совпадает с `package.json`;
- в ссылке нет двух версий;
- не осталось старых ручных токенов;
- команда `assets:stamp` существует.

## Граница решения

Токен добавляется к ресурсам, которые подключены непосредственно из HTML. Статические относительные импорты внутри ES-модулей массово не переписываются: это сделало бы исходники и валидаторы избыточно хрупкими. Целостность полного графа модулей подтверждают `validate:assets` и настоящий `browser-smoke`.

## Правило выпуска

Нельзя менять версию вручную только в отдельных URL. Источник версии — `package.json`; ссылки обновляются командой `npm run assets:stamp`.

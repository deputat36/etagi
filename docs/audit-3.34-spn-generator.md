# Полный аудит генератора расклеек СПН

Дата аудита: версия 3.34.0.

## Краткий вывод

Генератор уже решает основную практическую задачу: помогает СПН быстро собрать расклейку, выбрать сценарий, заполнить данные, проверить смысл, распечатать, провести тест и обработать отклики.

Главная сильная сторона — инструмент развивается не как дизайнерский редактор, а как рабочий помощник СПН: от макета до анализа результата.

Главная проблема — интерфейс стал функциональным, но перегруженным. Новичку может быть сложно понять, какие блоки обязательны, а какие нужны только после печати или для анализа.

## Что уже хорошо

1. Быстрый сценарный вход

СПН может выбрать рабочую ситуацию: собственник, покупатель, подъезд, объект, консультация или нестандартный макет. Это лучше, чем заставлять искать шаблон вручную.

2. Продающие шаблоны

Добавлен отдельный пакет продающих шаблонов. Они построены вокруг смысла: повод остановиться, выгода, доверие, призыв и контакт.

3. Проверка практической готовности

Инструмент проверяет не только технические ошибки, но и продающие элементы: заголовок, контекст, призыв, причины откликнуться и снятие опасений.

4. Работа после печати

Генератор теперь закрывает весь цикл: макет → печать → расклейка → звонок → квалификация → итог теста.

5. Гибкость

Есть управление составом, порядком блоков, печатными форматами, фото, QR, отрывными телефонами, фирменностью и плотностью.

## Главные проблемы

1. Перегруженность интерфейса

На одной странице одновременно появляются настройки макета, фото, печать, качество, план расклейки, скрипт, квалификация, итог теста, сохранение и помощь. Для опытного пользователя это полезно, для новичка — тяжело.

2. Нет явного разделения: «сделать макет» и «работать после расклейки»

Блоки анализа, квалификации и результата нужны, но они не должны мешать человеку, который сейчас просто хочет напечатать лист.

3. Сценарные фильтры ещё не полностью соответствуют новым пакетам

Есть быстрый мастер, но внутренние сценарии фильтрации пока не выделяют отдельно: продающие, доверие, A/B, пустые шаблоны, Борисоглебск.

4. Мало защиты от плохого визуального результата

Есть проверка переполнения и качества, но нет автоматической визуальной оптимизации текста: например, уменьшить описание, поднять телефон, выключить лишний блок для компактной печати.

5. Нет полноценного режима новичка

Нужно скрывать второстепенные настройки и вести СПН по шагам: ситуация → шаблон → контакты → объект → печать.

6. Нет сохранения результатов тестов

Есть калькулятор результата и копирование, но пока нет локального журнала тестов по макетам.

## План улучшений

### Этап 1. Упростить вход и снизить перегрузку

- Добавить явные режимы интерфейса: `Быстрый` и `Расширенный`.
- В быстром режиме показывать только обязательные блоки.
- Блоки после расклейки свернуть в отдельный раздел `После расклейки`.
- Сохранение и помощь оставить ниже, но сделать компактнее.

### Этап 2. Усилить продающий результат

- Добавить больше продающих шаблонов по реальным ситуациям СПН.
- Сделать отдельные фильтры: `Продающие`, `Доверие`, `A/B`, `Пустые`, `Борисоглебск`.
- Добавить подсказки по заголовкам: цена, спрос, конкретный дом, безопасная сделка, консультация.

### Этап 3. Автоматическая оптимизация макета

- Улучшить кнопку `Подстроить автоматически`.
- Если макет перегружен — предлагать выключить фото, QR, доп. блок или уменьшить количество макетов на А4.
- Если нет фото — перераспределять пространство без пустот.
- Для отрывных телефонов сохранять читаемость даже в компактном режиме.

### Этап 4. Журнал тестов

- Добавить локальное сохранение итогов тестовой расклейки.
- Показывать историю: дата, сценарий, листы, отклики, горячие, тёплые, вывод.
- Дать возможность скопировать сводку по нескольким тестам.

### Этап 5. Проверка перед печатью

- Сделать финальную кнопку `Готово к печати`.
- Перед печатью показывать короткий список: телефон, заголовок, контекст, читаемость, отрывные, линии реза.
- Если есть критичные ошибки — не мешать печатать, но явно предупреждать.

## Что уже начато после аудита

Начат этап 1: добавлен `Маршрут СПН`.

Маршрут показывает прогресс подготовки макета:

- ситуация;
- контакт;
- контекст;
- текст;
- доверие.

Также маршрут показывает следующий шаг и кнопку действия. Это снижает перегрузку и ведёт СПН по понятной цепочке.

## Следующий самостоятельный шаг

Следующим шагом нужно сделать режим `Быстрый / Расширенный`:

- быстрый режим — только то, что нужно для печати;
- расширенный режим — все настройки, аналитика, журнал и дополнительные инструменты.

Это должно стать самым важным улучшением удобства интерфейса.

# Актуальный аудит и план развития генератора расклеек СПН

Дата аудита: 30.06.2026
Репозиторий: `deputat36/etagi`
Текущая версия проекта: `3.84.0`

## 1. Назначение проекта

Проект — статический генератор печатных макетов для специалистов по недвижимости компании «Этажи» в городе Борисоглебск.

Главная задача проекта — не просто сверстать объявление, а помочь СПН быстро пройти рабочий сценарий:

1. Понять задачу расклейки.
2. Выбрать подходящий шаблон.
3. Подставить свои контакты и данные объекта/района.
4. Проверить читаемость, телефон, QR, плотность и смысл объявления.
5. Распечатать макет.
6. Сформировать задание на расклейку.
7. Зафиксировать результат после расклейки.

Инструмент должен быть полезен не только опытным СПН, но и новичкам, которым сложно самостоятельно выбрать формат, текст, плотность, количество макетов на А4 и правильный призыв.

## 3. Главные проблемы текущей версии

### 3.1. Интерфейс богатый, но может быть сложным для обычного СПН

В генераторе много возможностей: блоки, QR, фото, печать, сохранения, отчёты, быстрые исправления, доп. настройки. Для опытного пользователя это плюс, но новичок может потеряться.

Нужно сильнее разделить интерфейс на два режима:

- `Новичок` — только обязательные действия и безопасные подсказки.
- `Расширенно` — полный конструктор для опытного пользователя.

### 3.2. Быстрый подбор должен стать главным входом

Сейчас быстрый подбор ситуации существует, но воспринимается как дополнительный блок. Его нужно сделать основным маршрутом:

1. Что делаем сегодня?
2. Где клеим?
3. Кому должен позвонить человек?
4. Какой формат печати подходит?
5. Что отслеживаем после расклейки?

### 3.3. Шаблоны нужно классифицировать по офисной пользе

Сейчас шаблоны имеют цель, теги, плотность и другие параметры. Для внедрения в офисе нужны дополнительные признаки:

- рекомендовано офисом;
- подходит новичку;
- требует проверки менеджером;
- сценарий: подъезд, район, объект, новостройка, услуга, личный бренд;
- риск: низкий, средний, высокий;
- комментарий менеджера.

### 3.4. Нужен режим менеджера

Менеджеру отдела продаж нужен не дизайнерский редактор, а контроль качества активности:

- какие макеты рекомендовать новичкам;
- какие шаблоны использовать по собственникам;
- какие шаблоны использовать по ЖК «Теллерманов сад»;
- как проверить макет перед печатью;
- как сравнить результаты расклейки;
- когда повторить расклейку, а когда заменить текст.

### 3.5. `app.js` остаётся слишком центральным файлом

Файл управляет формой, шаблонами, сохранениями, профилем, печатью, блоками и контролем качества. Это допустимо для текущей версии, но усложняет дальнейшие безопасные изменения.

Нужно постепенно выносить логику в отдельные контроллеры без резкого переписывания проекта.

### 3.6. Основной CSS трудно сопровождать

`assets/css/app.css` сильно сжат. Это не мешает работе сайта, но ухудшает поддержку. При следующих визуальных изменениях его нужно отформатировать или постепенно разнести по логическим файлам.

## 4. Целевая концепция

Генератор должен стать офисным помощником СПН по расклейке, а не просто редактором объявления.

Правильная формула продукта:

```text
Ситуация СПН → рекомендованный шаблон → безопасная проверка → печать → задание → отчёт → вывод по эффективности
```

## 5. Приоритетный план улучшений

### Этап 1. Офисное внедрение и документация

Статус: можно делать автономно.

Задачи:

1. Обновить этот аудит под актуальную версию проекта.
2. Создать план внедрения в офисе: `docs/office-rollout-plan.md`.
3. Создать инструкцию СПН на одну страницу: `docs/spn-one-page-instruction.md`.
4. Создать чек-лист менеджера перед печатью: `docs/manager-print-checklist.md`.
5. Добавить новые документы в README.
6. Не менять версию, если изменения только документационные.

Критерий готовности:

- новый сотрудник может открыть инструкцию и понять, как сделать макет;
- менеджер может проверить макет по чек-листу;
- понятно, как внедрять генератор в отделе продаж.

### Этап 2. Главный экран «Что делаем сегодня?»

Статус: можно делать автономно, если не меняется брендовая политика и юридические обещания.

Задачи:

1. Перестроить быстрый подбор так, чтобы он был главным входом.
2. Добавить ситуации:
   - найти собственника;
   - есть покупатель;
   - продать объект;
   - новостройка / ЖК «Теллерманов сад»;
   - консультация / безопасность;
   - личный бренд СПН;
   - нестандартный макет.
3. Для каждой ситуации показывать:
   - рекомендуемый формат печати;
   - главный смысл объявления;
   - что проверить перед печатью;
   - что считать после расклейки.
4. Не ломать существующий поиск по шаблонам.

Критерий готовности:

- СПН начинает не с поиска шаблона, а с выбора реальной рабочей задачи.

### Этап 3. Режим «Новичок СПН»

Статус: можно делать автономно.

Задачи:

1. Добавить третий режим интерфейса или переосмыслить текущий быстрый режим:
   - новичок;
   - быстро;
   - расширенно.
2. В режиме новичка оставить только:
   - выбор ситуации;
   - выбор шаблона;
   - имя и телефон;
   - район/адрес/ЖК;
   - цена/бюджет/параметры;
   - текст объявления;
   - проверка;
   - печать.
3. Скрыть сложные настройки:
   - порядок блоков;
   - поля;
   - радиус;
   - ручная плотность;
   - масштаб заголовка;
   - масштаб телефона;
   - JSON-импорт/экспорт;
   - расширенная аналитика.
4. Добавить подсказку: «Начните с рекомендованного шаблона, потом правьте только выделенные поля».

Критерий готовности:

- новичок может сделать первый макет без объяснений менеджера.

### Этап 4. Офисные признаки шаблонов

Статус: можно делать автономно в формате, совместимом со старой структурой JSON.

Задачи:

1. Добавить необязательный объект `office` в шаблоны:

```json
"office": {
  "recommended": true,
  "level": "newbie",
  "scenario": "entrance",
  "risk": "low",
  "managerNote": "Подходит для поиска собственников в конкретном доме"
}
```

2. Обновить валидатор шаблонов, чтобы он принимал `office`.
3. Добавить фильтр «Рекомендовано офисом».
4. Добавить визуальную метку «Для новичка».
5. Добавить визуальную метку «Проверить с менеджером» для рискованных макетов.

Критерий готовности:

- менеджер может заранее выделить безопасные шаблоны для отдела.

### Этап 5. Расширение локальной библиотеки Борисоглебска

Статус: можно делать автономно на основе уже известных сценариев, без публикации закрытых данных.

Задачи:

1. Добавить пакет шаблонов для собственников:
   - узнать цену;
   - есть покупатель;
   - продажа без публикации;
   - квартира в вашем доме;
   - дом/участок в районе;
   - безопасная продажа.
2. Добавить пакет шаблонов для ЖК «Теллерманов сад» без закрытой информации:
   - старт продаж;
   - семейная ипотека;
   - материнский капитал;
   - молодая семья;
   - подбор планировки;
   - квартиры в новом доме.
3. Добавить пакет для услуг:
   - оценка недвижимости;
   - проверка документов;
   - ипотечная консультация;
   - безопасная сделка;
   - продажа и покупка одновременно.
4. Добавить офисные теги и сценарии.

Критерий готовности:

- у СПН есть готовые варианты под основные рабочие ситуации офиса.

### Этап 6. Чек-лист менеджера внутри интерфейса

Статус: можно делать автономно.

Задачи:

1. Добавить компактный блок «Проверка менеджера» рядом с контролем качества или в help.
2. Проверять:
   - телефон;
   - читаемость;
   - канал отклика;
   - снятие опасений;
   - понятный район/объект;
   - перегруз текста;
   - QR;
   - отрывные телефоны;
   - формат печати.
3. Дать кнопку «Скопировать замечания менеджера».

Критерий готовности:

- менеджер может быстро дать обратную связь СПН по макету.

### Этап 7. История расклеек и простая аналитика

Статус: можно делать автономно, если всё хранится локально в браузере.

Задачи:

1. Сохранять историю отчётов после расклейки.
2. Добавить список последних отчётов.
3. Добавить повтор задания из отчёта.
4. Добавить экспорт CSV.
5. Добавить простые выводы:
   - отклика нет — изменить заголовок или место;
   - звонки есть, целевых мало — уточнить текст;
   - хороший отклик — повторить в похожих домах.

Критерий готовности:

- расклейка становится измеримой активностью, а не разовым действием.

### Этап 8. Техническая декомпозиция

Статус: делать постепенно, маленькими безопасными шагами.

Задачи:

1. Вынести работу с формой из `app.js`.
2. Вынести работу с шаблонами из `app.js`.
3. Вынести печатный сценарий из `app.js`.
4. Вынести сохранения интерфейса из `app.js`.
5. Оставить `app.js` точкой сборки.
6. После каждого шага запускать `npm run validate`.

Критерий готовности:

- новые функции добавляются без риска сломать базовый сценарий.

1. Меняет юридические обещания компании.
2. Использует закрытые данные по новостройкам, адресам, ценам или застройщику.
3. Меняет фирменный стиль компании «Этажи» принципиально.
4. Добавляет сбор персональных данных на сервер.
5. Подключает внешние сервисы.
6. Удаляет существующую функцию.
7. Меняет поведение печати так, что старые макеты могут распечататься иначе.
8. Делает публичные заявления о конкретных условиях ипотеки без проверки актуальности.

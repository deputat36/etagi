# Инвентаризация шаблонов Борисоглебска

Дата: 10 июля 2026 года.

## Итог

Локальная библиотека состоит из двух пакетов:

```text
data/templates_borisoglebsk.json
 data/templates_borisoglebsk_expanded.json
```

Общее количество локальных шаблонов:

```text
21
```

Основной пакет содержит 9 ранее работающих сценариев. Расширяющий пакет добавляет 12 новых сценариев без изменения старых шаблонов.

## Сценарии для новичка

Новичку разрешены мягкие сценарии без сложных юридических и рекламных обещаний:

- центр;
- Аэродромная;
- Северный район;
- квартира для семьи;
- предварительный интерес к новостройке;
- оценка недвижимости;
- планируете продавать в этом доме;
- подбор аренды;
- подбор дома с участком.

Перед печатью обязательно проверить телефон, район и отсутствие выдуманного готового покупателя.

## Сценарии с проверкой менеджера

Проверка менеджера обязательна для сценариев:

- объект с ценой и фотографией;
- дом и земельный участок;
- коммерческое помещение;
- наследство;
- срочная продажа;
- альтернативная сделка;
- переход во вторичное или новое жильё;
- документы перед продажей;
- помощь с арендой собственнику;
- частное объявление без бренда.

Причины:

- юридические нюансы;
- риск неподтверждённых обещаний;
- необходимость точных характеристик объекта;
- условия комиссии и услуги;
- использование частного или объектного позиционирования.

## Запрещённые формулировки

В локальных шаблонах нельзя использовать без подтверждения:

```text
готовый покупатель
покупатель уже есть
гарантируем продажу
продадим за 7 дней
продадим за неделю
точная цена
ипотека гарантирована
одобрение гарантировано
100% результат
```

Допустимый подход:

```text
оценим ситуацию
составим реалистичный план
подберём варианты
проверим информацию
обсудим условия
подключим профильного специалиста
```

## Автоматическая проверка

Покрытие контролируется командой:

```bash
npm run validate:borisoglebsk-coverage
```

Проверка контролирует:

- оба локальных пакета подключены в `assets/js/templates.js`;
- количество шаблонов не меньше 20;
- присутствуют обязательные `templateId`;
- office-сценарии не повторяются;
- цели имеют минимальное покрытие;
- высокий риск используется только с уровнем `manager`;
- теги соответствуют office-метаданным;
- отсутствуют неподтверждённые гарантии и обещания.

## Правило дальнейшего расширения

Новый локальный шаблон должен отвечать на конкретную рабочую ситуацию СПН, а не просто менять заголовок существующего объявления.

Перед добавлением нужно определить:

1. Для кого шаблон.
2. Какую задачу он решает.
3. Какой риск формулировок.
4. Может ли использовать новичок.
5. Нужна ли проверка менеджера.
6. Какой формат А4 рекомендуется.
7. Какие данные обязательно заменить перед печатью.

Шаблоны с одинаковым смыслом и разными декоративными формулировками не добавляются в основной офисный набор.

# История изменений

## 3.84.0

- Усилена проверка телефона в `assets/js/quality.js`.
- Контроль качества теперь считает критичной ошибкой номер, который выглядит неполным или ошибочным при включённых контактах или отрывных телефонах.
- Добавлена подсказка, если в поле телефона есть лишний текст, ухудшающий читаемость отрывных листков.
- `assets/js/preprintSummary.js` теперь отдельно предупреждает о неполном номере перед печатью.
- Добавлены безопасные помощники для QR и фото: включённые блоки не отключаются автоматически, а переводят пользователя к нужному полю.
- Подавленный QR-дубль больше не учитывается в фильтре, сводке, приоритете и состоянии кнопки печати.
- Добавлены проверки `validate:photo-intent-action`, `validate:suppressed-quality-items` и `validate:quality-helper-imports`.
- Версия состояния и пакета обновлена до `3.84.0`.

## 3.83.0

- Обновлён `README.md` под текущее состояние проекта.
- В структуру добавлены `assets/js/layoutExtras.js`, `assets/js/qualityExtraActions.js` и все актуальные `tools/validate-*.mjs` проверки.
- В описание возможностей добавлены единый модуль расширенных полей, быстрые исправления контроля качества и расширенный набор проверок проекта.
- Раздел проверки проекта теперь перечисляет все отдельные `npm run validate:*` команды.
- Версия состояния и пакета обновлена до `3.83.0`.

## 3.82.0

- Добавлена проверка `tools/validate-layout-extras.mjs` для расширенных полей макета.
- Проверка контролирует, что ключи хранения призыва, подписи отрывных листков и брендовой строки описаны только в `assets/js/layoutExtras.js`.
- `npm run validate` теперь запускает проверку `validate:layout-extras` вместе с остальными проверками проекта.
- Это защищает проект от повторного размножения одинаковых `localStorage`-ключей по разным JS-файлам.
- Версия состояния и пакета обновлена до `3.82.0`.

## 3.81.0

- В `assets/js/layoutExtras.js` добавлена общая функция записи расширенных полей макета.
- Редакторы `spnContactEditor.js`, `spnTearOffEditor.js` и `spnBrandEditor.js` больше не хранят собственные ключи `localStorage` для призыва, подписи и брендовой строки.
- `assets/js/qualityExtraActions.js` теперь использует общий helper для быстрых исправлений и сохраняет прежнее поведение: поле раскрывается, получает фокус и макет обновляется.
- Это завершает перенос записи и чтения расширенных полей макета в один общий модуль.
- Версия состояния и пакета обновлена до `3.81.0`.

## 3.80.0

- `assets/js/layoutExtrasSync.js` переведён на общий модуль `assets/js/layoutExtras.js`.
- `assets/js/preprintSummary.js` теперь берёт призыв, подпись отрывных листков и брендовую строку через общий модуль.
- В `assets/js/layoutExtras.js` добавлено чтение “сырого” значения без запасного текста, чтобы сводка перед печатью могла отличать пустое поле от дефолта.
- Это завершает объединение расширенных полей макета вокруг одного источника правил.
- Версия состояния и пакета обновлена до `3.80.0`.

## 3.79.0

- Добавлен общий модуль `assets/js/layoutExtras.js` для расширенных полей макета.
- Сохранение макетов, скачивание JSON, рендер предпросмотра и проверка качества теперь используют один общий источник правил для призыва, подписи отрывных листков и брендовой строки.
- Это снижает риск, что одно из мест будет показывать или сохранять старое значение после будущих доработок.
- Поведение существующих макетов сохранено: рендер по-прежнему не подменяет сохранённый текст дефолтным значением из видимого редактора.
- Версия состояния и пакета обновлена до `3.79.0`.

## 3.78.0

- Добавлена проверка `tools/validate-changelog.mjs` для истории изменений.
- Проверка контролирует, что версии в `docs/changelog.md` не дублируются.
- Проверка также контролирует, что версии идут сверху вниз по убыванию.
- `npm run validate` теперь запускает проверку changelog вместе с остальными проверками проекта.
- Версия состояния и пакета обновлена до `3.78.0`.

## 3.77.0

- Добавлена проверка `tools/validate-package-scripts.mjs` для команд проверки в `package.json`.
- Проверка контролирует, что каждый `validate:*` скрипт включён в общий `npm run validate`.
- Проверка также контролирует, что `validate:*` команды запускают существующие файлы `tools/*.mjs`.
- Это снижает риск добавить новую проверку и забыть подключить её к общей проверке проекта.
- Версия состояния и пакета обновлена до `3.77.0`.

## 3.76.0

- Добавлена проверка `tools/validate-version-sync.mjs` для синхронизации версии проекта.
- Проверка сравнивает версию в `package.json`, `assets/js/state.js` и верхнюю запись в `docs/changelog.md`.
- `npm run validate` теперь запускает проверку синхронизации версий вместе с остальными проверками.
- Это снижает риск выпустить обновление с разъехавшимися версиями в коде и документации.
- Версия состояния и пакета обновлена до `3.76.0`.

## 3.75.0

- Добавлена проверка `tools/validate-storage-safety.mjs` для защиты браузерных сохранений.
- Проверка контролирует, что важные операции `localStorage` в `assets/js/storage.js` имеют безопасные значения возврата при ошибках.
- Проверка также контролирует, что `assets/js/app.js` показывает пользователю понятные сообщения при ошибках сохранения профиля, макетов и избранного.
- `npm run validate` теперь запускает новую проверку вместе с остальными проверками проекта.
- Версия состояния и пакета обновлена до `3.75.0`.

## 3.74.0

- Усилена работа с профилем СПН, удалением сохранённых макетов и избранными шаблонами.
- `assets/js/storage.js` теперь безопасно обрабатывает ошибки `localStorage` для профиля, удаления макета и переключения избранного.
- `assets/js/app.js` показывает понятные статусы, если профиль, удаление макета или избранное не удалось сохранить из-за ограничений браузера.
- Это продолжает защиту интерфейса от повреждённых или переполненных браузерных сохранений.
- Версия состояния и пакета обновлена до `3.74.0`.

## 3.73.0

- Улучшена обработка ошибок сохранения макетов в `assets/js/app.js` и `assets/js/storage.js`.
- Кнопка «сохранить последний макет» теперь показывает понятное сообщение, если браузер не смог сохранить данные.
- Именованные макеты теперь не ломают интерфейс при ошибке `localStorage`; пользователь получает статус о невозможности сохранить макет.
- Сохранение именованного макета стало устойчивее к старым или повреждённым записям без имени.
- Версия состояния и пакета обновлена до `3.73.0`.

## 3.72.0

- Усилена работа с последним сохранённым макетом в `assets/js/storage.js`.
- Сохранение последнего макета теперь защищено от ошибок `localStorage` и возвращает признак успеха.
- Загрузка последнего макета теперь безопасно обрабатывает повреждённые данные в браузере и возвращает пустой результат вместо сбоя интерфейса.
- Версия состояния и пакета обновлена до `3.72.0`.

## 3.71.0

- Улучшена загрузка сохранённых макетов в `assets/js/app.js`.
- Последний сохранённый макет и выбранный сохранённый макет теперь загружаются от чистого базового состояния.
- Это предотвращает смешивание старых данных текущего макета с неполными сохранениями из старых версий.
- Импорт JSON переиспользует общий безопасный путь загрузки состояния.
- Версия состояния и пакета обновлена до `3.71.0`.

## 3.70.0

- Улучшено открытие JSON-файлов макетов в `assets/js/app.js`.
- Файл теперь загружается от чистого базового состояния, а не накладывается поверх текущего макета.
- Это предотвращает смешивание старых данных объекта с импортированным макетом, если файл неполный или создан в старой версии.
- После открытия файла список сохранённых макетов визуально снимает выбор.
- Версия состояния и пакета обновлена до `3.70.0`.

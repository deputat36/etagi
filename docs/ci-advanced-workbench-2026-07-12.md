# Контрольный CI рабочего блока после печати

Служебный документ для запуска актуального `main`.

Проверяется:

```text
validate → browser-smoke → print-screenshot × 5 → collect-print-screenshots
```

Критерии:

- три существующие секции переносятся, а не клонируются;
- observer отключается после сборки;
- режимы `Быстро` и `Новичок` не получают пустой контейнер;
- runtime и печатные сценарии остаются стабильными.

PR служебный. Код уже находится в `main`.

## Рабочий путь

Генератор закрывает основной процесс отдела продаж:

```text
задача → сценарий → шаблон → данные → фото и QR → проверка качества → печать → задание → расклейка → отчёт → аналитика
```

Реализованы:

- режимы `Новичок`, `Быстро` и `Расширенно`;
- Wizard Flow с восемью шагами и итогами `До печати / После печати`;
- закреплённый переход к штатной проверке печати;
- печать 1, 2 и 4 макетов на A4;
- фото-компоновки `photo_left`, `photo_card`, `newbuild_visual`, `agent_brand_photo`;
- QR, крупный телефон, отрывные полосы, линии реза и экономный цвет;
- задание на расклейку, отчёт, история, CSV и менеджерская сводка;
- мобильный режим исполнителя с точками, планом и фактом, статусами и проблемами;
- полный локальный backup рабочего пространства с режимами `merge / replace` и откатом при ошибке.

## Закрытые технические риски аудита от 11 июля

### Уникальность шаблонов

- введён реестр пакетных алиасов;
- итоговые `templateId` уникальны;
- неразрешённых итоговых дублей: 0;
- вероятных смысловых дублей: 0;
- инвентаризация защищена проверкой свежести.

### Runtime и browser smoke

- ранние `error` и `unhandledrejection` собираются до загрузки интерфейса;
- browser smoke проверяет накопленные ранние ошибки;
- переход `private → agent_brand_photo` нормализован и защищён регрессией;
- helper-модули подключаются через единую runtime-точку;
- failure-логи и screenshot-artifacts наблюдаемы в CI.

### Сохранность данных

Backup включает профиль СПН, автосохранение, последний макет, именованные макеты, избранное, UI-настройки, настройки мастера и историю отчётов. Импорт предварительно проверяется и не должен частично портить рабочее пространство.

### Полевое выполнение

Исполнитель может работать с телефона, отмечать до 30 точек, статусы, план и факт листов, проблемы и наличие фотоотчёта без обязательной передачи изображений на сервер.

## Автоматическая проверка

Workflow `.github/workflows/validate.yml` запускается для изменений `index.html`, `assets`, `data`, `help`, `docs`, `tools`, `README.md`, `package.json` и самого workflow.

Он выполняет:

1. `npm run validate`;
2. настоящий browser smoke;
3. пять независимых print-screenshot сценариев;
4. сбор итогового artifact `print-screenshots`.

Последний полный запуск, зафиксированный в релиз-кандидате: workflow run #1390 от 14 июля 2026 года.

После run #1390 release gate был усилен проверкой viewport-приёмки и evidence-пакета. Поэтому новые изменения нельзя считать подтверждёнными только на основании #1390: перед переводом в `READY` нужен новый успешный полный workflow run.

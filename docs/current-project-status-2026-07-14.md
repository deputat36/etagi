## Рабочий путь

Генератор закрывает основной процесс отдела продаж:

```text
задача → сценарий → шаблон → данные → фото и QR → проверка качества → печать → задание → расклейка → отчёт → аналитика
```

Реализованы:

- режимы `Новичок`, `Быстро` и `Расширенно`;
- Wizard Flow с восемью шагами;
- официальный набор `1, 2, 3, 4, 6 и 8 макетов на A4`;
- запрет импорта неподдерживаемых 10/12;
- фото-компоновки `photo_left`, `photo_card`, `newbuild_visual`, `agent_brand_photo`;
- QR, крупный телефон, линии реза и экономный цвет;
- задание, мобильное выполнение, отчёт, история, CSV и аналитика;
- полный локальный backup рабочего пространства.

## Закрытые технические риски

### Интерфейс и действия

- создан machine-readable реестр основных действий;
- browser smoke покрывает сохранение, загрузку, очистку, JSON, печатный диалог и клавиатуру;
- failure artifact содержит последнее действие и снимок состояния;
- карточки шаблонов и inline-редактор имеют управляемый Tab-путь;
- команда `Вписать` рассчитывает масштаб по реальной рабочей области.

### Сохранность данных

- резерв создаётся перед очисткой, импортом и заменой макета;
- последнее разрушительное действие можно отменить;
- удаление именованного макета требует подтверждения;
- последний удалённый макет можно восстановить;
- конфликт названия не перезаписывает соседнюю запись;
- JSON имеет версию, схему и понятную диагностику;
- повторный импорт того же файла разрешён.

### Печатная матрица

Официальные форматы:

```text
1 / 2 / 3 / 4 / 6 / 8 на A4
```

Автоматически подтверждено:

- точное количество карточек;
- полный телефон и имя СПН;
- отсутствие `overflow`;
- карточки и контакты внутри границ;
- восемь отрывных полос для 1/2/3/4;
- отключение и блокировка полос для 6/8;
- минимальные размеры телефона 30/28/24/20/18/16 px;
- отдельные screenshot-сценарии для 3, 6 и 8;
- компактные профили для 3 и 8 после подтверждённого overflow.

Подробности:

```text
docs/print-format-policy.md
docs/print-format-coverage.md
docs/print-screenshot-regression.md
```

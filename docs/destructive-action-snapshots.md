# Снимки и отмена разрушительных действий

Генератор сохраняет одну последнюю резервную точку непосредственно перед действием, которое может заменить или удалить текущую работу. После этого специалист может вернуть состояние одной кнопкой.

## Когда создаётся снимок

Снимок создаётся перед:

- очисткой данных объекта;
- загрузкой выбранного именованного макета;
- загрузкой последнего макета из браузера;
- успешным импортом JSON-макета.

Ошибочный JSON не перезаписывает существующую резервную точку. Для загрузки именованного или последнего макета снимок не создаётся, если фактически нечего загружать.

## Что входит в снимок

Ключ хранения:

```text
etagi-raskleyka-destructive-snapshot-v1
```

В снимок входят:

- версия формата, уникальный идентификатор, дата и причина;
- все поля формы и переключатели состава макета;
- профиль СПН;
- служебные подписи контактов, бренда и отрывных полос;
- активная задача, шаблон, режим компоновки и количество копий;
- порядок блоков;
- QR;
- отображаемые фотографии в виде data URL;
- выбранный именованный макет и рабочий сценарий.

Сначала браузер пытается сохранить полный снимок. Если лимит localStorage не позволяет сохранить тяжёлые фотографии, сохраняется облегчённый вариант без фото с `photosOmitted: true`. Остальные данные не теряются.

## Кнопка отмены

Рядом с действиями профиля отображается кнопка `Отменить последнее действие`.

- без резервной точки кнопка отключена;
- после разрушительного действия название уточняет, что именно будет отменено;
- подсказка показывает причину и время снимка;
- облегчённый снимок явно помечается как вариант без фотографий;
- восстановление проходит через штатную проверку JSON-макета;
- выбранный именованный макет и рабочий сценарий возвращаются, если они ещё доступны;
- использованный снимок удаляется только после успешного восстановления;
- при ошибке или тайм-ауте снимок сохраняется для повторной попытки.

Отмена одноразовая. Повторного шага `Вернуть отменённое` пока нет. Новый успешный разрушительный сценарий заменяет предыдущую резервную точку.

## Проверка

Статический контракт входит в общую проверку безопасности браузерного хранения:

```bash
npm run validate:storage-safety
npm run smoke:ui-actions
```

Сценарий `Destructive snapshot smoke` проверяет создание резервной точки перед очисткой, именованной загрузкой, загрузкой последнего макета и успешным импортом. Он также проверяет профиль, QR, служебную подпись, порядок блоков, количество копий, фото и защиту снимка от ошибочного импорта.

Сценарий `Destructive undo smoke` выполняет реальное восстановление после очистки, загрузки именованного макета, загрузки последнего макета и импорта JSON. После успешной отмены он требует восстановления данных и удаления использованной резервной точки.

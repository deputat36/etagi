# План развития генератора расклеек

Цель проекта — сделать простой и быстрый инструмент для СПН: открыл ссылку, выбрал задачу, выбрал шаблон, заменил данные и распечатал.

## Принципы

1. Сначала удобство СПН, потом сложные настройки.
2. Готовые шаблоны должны решать реальные задачи: куплю, продам, услуги, новостройки, аренда, личный бренд.
3. Основной сценарий должен занимать 1–2 минуты.
4. Все сложные настройки должны быть спрятаны в расширенный режим.
5. Проект должен оставаться статическим, чтобы работать на GitHub Pages без сервера и базы данных.

## Ближайшие этапы

### Этап 1. Проверка базы

- Проверить запуск на GitHub Pages.
- Проверить загрузку `data/templates.json`.
- Проверить печать 1, 2 и 4 макетов на А4.
- Проверить мобильный и обычный экран.
- Проверить, что проект не конфликтует с другими репозиториями.

### Этап 2. Удобство СПН

- Добавить быстрый профиль специалиста: имя, телефон, район, должность.
- Добавить быстрые кнопки: квартира, дом, участок, новостройка, аренда.
- Добавить быстрые районы Борисоглебска.
- Добавить подсказки по заполнению полей.
- Сделать кнопку «очистить только объект», не стирая данные СПН.

### Этап 3. Шаблоны

- Расширить библиотеку шаблонов до 80–120 вариантов.
- Разделить шаблоны по типам: плотные, обычные, полупустые.
- Добавить больше шаблонов с фото и двумя фото.
- Добавить отдельные шаблоны под собственников, покупателей, новостройки, услуги и личный бренд.

### Этап 4. Качество печати

- Улучшить контроль переполнения.
- Добавить автоматическое уменьшение текста.
- Добавить режим экономной печати.
- Добавить нормальные поля под резку.
- Добавить предпросмотр линий разреза.

### Этап 5. Сохранение и перенос

- Сохранять профиль СПН отдельно от макета.
- Добавить экспорт/импорт профиля.
- Добавить экспорт/импорт готового макета.
- Добавить несколько локальных сохранений.

## Что пока не делаем

- Серверную часть.
- Авторизацию.
- Базу данных.
- Загрузку фото на сервер.
- Общую статистику по СПН.

Это можно добавить позже, если инструмент приживется в офисе.

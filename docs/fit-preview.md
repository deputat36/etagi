# Вычисляемое «Вписать»

Кнопка `Вписать` подбирает масштаб предпросмотра A4 по фактическому свободному месту в рабочей области.

## Как рассчитывается масштаб

Расчёт использует одновременно:

- доступную ширину `.sheet-wrap`;
- доступную высоту `.sheet-wrap`;
- внутренние отступы рабочей области;
- физическую ширину и высоту листа `#printSheet` до transform;
- небольшой безопасный запас от округления браузером.

Выбирается меньший коэффициент по ширине и высоте. Масштаб ограничен диапазоном 20–100%, округляется вниз и записывается в ползунок `#zoom`.

## Поведение режима

После нажатия `Вписать`:

- кнопка получает `aria-pressed="true"`;
- ползунок показывает рассчитанное значение;
- рядом выводится текущее значение в процентах;
- лист не выходит за доступную ширину или высоту рабочей области;
- при изменении размеров окна или контейнера масштаб пересчитывается автоматически.

Для отслеживания фактического размера используется `ResizeObserver`. Обработчик `window.resize` остаётся резервным механизмом.

Если специалист вручную двигает ползунок, автоматическое вписывание отключается, `aria-pressed` возвращается в `false`, а последующие изменения окна не заменяют выбранный вручную масштаб. Повторное нажатие `Вписать` снова включает автоматический режим.

## Проверка

Статический контракт:

```bash
npm run validate:fit-preview
```

Browser smoke:

```bash
npm run smoke:ui-actions
```

Сценарий `Fit preview smoke` проверяет формулу, фактическое отсутствие переполнения, пересчёт после уменьшения высоты окна, ручное переопределение и повторное включение режима.

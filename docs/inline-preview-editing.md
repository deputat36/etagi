# Редактирование текста прямо на макете

Предпросмотр А4 поддерживает быстрое редактирование основных текстовых блоков без обязательного поиска соответствующего поля в длинной форме.

## Как использовать

1. Нажать на текст в макете.
2. Изменить его непосредственно в предпросмотре.
3. Кликнуть вне редактируемого блока.
4. Значение синхронизируется с полем слева и обновится во всех копиях макета на листе.

Для коротких однострочных полей Enter завершает редактирование. Для многострочного текста используется Ctrl+Enter или клик вне блока. Esc отменяет текущее изменение.

## Клавиатурный порядок

В клавиатурный Tab-порядок входит только первая копия макета на листе. Это не заставляет пользователя повторно проходить одинаковые заголовки, телефоны и описания во второй, третьей и последующих копиях.

Остальные копии остаются редактируемыми кликом или касанием. Изменение любой копии по-прежнему синхронизируется с полем слева и обновляет весь лист.

## Какие блоки доступны

- заголовок;
- цена или бюджет;
- описание;
- район, тип объекта и параметры;
- отдельные преимущества;
- название и текст дополнительного блока;
- телефон и имя специалиста;
- призыв в контактном блоке;
- подпись QR;
- имя в компоновке `agent_brand_photo`.

Поля в левой панели остаются основным источником данных и запасным способом редактирования. Редактор на макете не создаёт отдельную копию текста.

Предупреждение о слишком длинной QR-ссылке не является частью подписи. Оно выводится отдельным служебным текстом, не редактируется и не записывается в поле `qrCaption`.

## Безопасность ввода

Вставка выполняется как обычный текст без HTML-разметки. После завершения редактирования запускается штатный рендер и контроль качества. Изменение одной копии на листе применяется ко всем одинаковым копиям.

## Печать

Подсветка редактируемых областей и служебная подсказка скрываются в печати и PDF. Атрибуты редактирования не меняют содержимое печатного листа.

## Проверка

Команда:

```bash
npm run test:inline-preview-editing
```

проверяет runtime-подключение, разметку редактируемых блоков, синхронизацию с формой, отмену через Esc, вставку только обычного текста, screenshot-сценарий прямого редактирования заголовка, browser smoke подписи при слишком длинной QR-ссылке и Tab-порядок для нескольких копий макета.

Эта проверка также запускается внутри `npm run validate:ink-efficiency`, поэтому входит в общий `npm run validate`.

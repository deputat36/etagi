# Формат JSON-макета

Отдельный файл макета используется для переноса одной листовки между браузерами и компьютерами. Он не является полным backup рабочего пространства и не содержит сохранённые списки, отчёты или настройки интерфейса.

## Текущая схема

Machine-readable схема находится в файле:

```text
data/layout-file.schema.json
```

Текущая версия:

```text
kind: etagi-raskleyka-layout
schemaVersion: 1
```

Формат плоский. Служебные поля находятся рядом с состоянием макета:

```json
{
  "kind": "etagi-raskleyka-layout",
  "schemaVersion": 1,
  "appVersion": "3.85.0",
  "exportedAt": "2026-07-20T12:00:00.000Z",
  "headline": "Продам квартиру",
  "printCount": 4
}
```

Полный файл содержит все обязательные поля состояния, включая видимость блоков, порядок блоков, настройки печати, фото и QR.

## Проверка перед импортом

До замены текущего макета проверяются:

- корректность JSON;
- объект в корне файла;
- идентификатор `kind`;
- поддерживаемая версия `schemaVersion`;
- наличие обязательных полей;
- типы строк, чисел и переключателей;
- допустимые значения режимов;
- диапазоны числовых настроек;
- состав и уникальность `blockOrder`;
- отсутствие неизвестных полей в формате v1.

При ошибке текущий макет не изменяется. В строке состояния показывается конкретная причина, а фокус возвращается к кнопке открытия файла.

## Совместимость

Файлы версии 3.85.0 и более ранние могли содержать только необёрнутый объект состояния. Такой файл считается старым форматом:

- известные поля проверяются;
- отсутствующие поля дополняются безопасными значениями по умолчанию;
- неизвестные поля не переносятся;
- после следующего скачивания файл сохраняется по схеме v1.

Неподдерживаемая будущая версия не открывается молча. Пользователь получает сообщение с номером версии файла и текущей поддерживаемой версией.

## Проверки

Статический контракт входит в общий домен безопасности хранения:

```bash
npm run validate:storage-safety
npm run smoke:ui-actions
```

Команда `validate:storage-safety` последовательно запускает прежние проверки localStorage и `tools/validate-layout-file-schema.mjs`.

Browser-сценарий `Layout file diagnostics smoke` проверяет экспорт v1, импорт старого формата и отказ для повреждённого JSON, массива вместо объекта, чужого типа файла, будущей версии и неверного типа поля. После каждого отказа состояние макета должно оставаться прежним.

# Формат JSON-макета

JSON-файл нужен для переноса одного макета между компьютерами и браузерами. Он не заменяет сохранённые макеты в браузере и не содержит историю работы.

## Новый формат

Экспортированный файл содержит состояние макета и четыре служебных поля:

```json
{
  "$schema": "data/layout-file.schema.json",
  "fileFormat": "etagi-raskleyka-layout",
  "fileFormatVersion": 1,
  "exportedAt": "2026-07-20T15:00:00.000Z"
}
```

Остальные поля описывают текст, компоновку, фото, QR, печать и видимость блоков. Полная машинно-читаемая схема находится в `data/layout-file.schema.json`.

## Совместимость

Старые плоские JSON-файлы без `fileFormat` продолжают открываться. При импорте генератор подставляет отсутствующие поля из безопасного состояния по умолчанию.

Новые неизвестные поля не ломают импорт: они игнорируются, а пользователь получает предупреждение. Известные поля с неправильным типом или неподдерживаемым значением отклоняются до изменения текущего макета.

## Понятные ошибки

Импорт различает:

- повреждённый JSON;
- список или текст вместо одного макета;
- файл другого приложения;
- слишком новую версию формата;
- посторонний JSON без признаков макета;
- неверный тип конкретного поля;
- неподдерживаемый формат печати, цветность, режим фото или компоновки.

При ошибке текущий макет не изменяется. Сообщение объясняет, что именно не так и какое действие требуется.

## Проверка

```bash
npm run validate:layout-file
npm run smoke:ui-actions
```

Первая команда проверяет схему, новый и старый форматы, ошибки типов, будущую версию и совместимость с неизвестными полями. Browser smoke подтверждает, что ошибочный импорт не заменяет текущий макет, а корректный файл восстанавливается без смешивания данных.

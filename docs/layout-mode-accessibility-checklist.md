# Чек-лист keyboard/focus для выбора компоновки

Проверять после изменений `state.js`, `app.js`, `spnUiMode.js`, `spnLayoutModeAccessibility.js` и стилей интерфейса.

## Базовая проверка

1. Открыть генератор.
2. Перейти к блоку «Быстрые режимы».
3. Нажать `Tab`, пока фокус не попадёт на выбранную компоновку.
4. Убедиться, что фокус имеет заметную синюю рамку.
5. Убедиться, что только одна кнопка имеет `tabindex="0"`.
6. Убедиться, что остальные кнопки имеют `tabindex="-1"`.
7. Убедиться, что выбранная кнопка имеет `aria-checked="true"`.

## Клавиатурная навигация

1. Нажать `ArrowRight` — должна выбраться следующая компоновка.
2. Нажать `ArrowDown` — должна выбраться следующая компоновка.
3. Нажать `ArrowLeft` — должна выбраться предыдущая компоновка.
4. Нажать `ArrowUp` — должна выбраться предыдущая компоновка.
5. На последней кнопке нажать `ArrowRight` — выбор должен перейти к первой.
6. На первой кнопке нажать `ArrowLeft` — выбор должен перейти к последней.
7. Нажать `Home` — должна открыться первая компоновка.
8. Нажать `End` — должна открыться последняя компоновка.

## Проверка результата

После каждого перехода убедиться, что:

- фокус остаётся на выбранной кнопке;
- класс `active` соответствует `aria-checked="true"`;
- предпросмотр А4 обновился;
- формат, фото, QR и телефон меняются только по правилам выбранной компоновки;
- страница не прокручивается стрелками вместо переключения кнопок;
- глобальные сочетания клавиш вне `#layoutModeGrid` работают обычно.

## Мобильная проверка

1. Открыть страницу при ширине до 520 px.
2. Убедиться, что подписи не обрезаются.
3. Убедиться, что кнопки имеют достаточную высоту для нажатия.
4. Проверить видимость `focus-visible` при подключённой физической клавиатуре.

## Ограничения

Helper работает только внутри `#layoutModeGrid`. Он не должен добавлять глобальный `keydown` на `document` или `window`.

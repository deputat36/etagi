# Читаемость важных подписей на мобильном экране

Проверка относится только к рабочему интерфейсу на ширине до 520 px. Печатный лист, данные пользователя и логика переходов мастера не меняются.

## Что увеличено

- статусы восьми шагов и итоги `До печати / После печати`;
- подписи количества макетов на А4, подсказки шагов и текущего действия;
- описание офисного сценария, отметка риска и статуса шаблона;
- подписи переключателя и badges компактного списка шаблонов.

На широком экране остаются прежние компактные размеры.

## Автоматическая проверка

```bash
npm run validate:mobile-readability
npm run smoke:browser
```

Browser smoke временно устанавливает iframe шириной 500 px и проверяет вычисленный `font-size` ключевых подписей. Затем ширина возвращается к 1280 px, и основной сценарий продолжается.

## Ручной чек-лист

Проверить ширины 360 px, 390 px и 500 px:

- [ ] В режиме «Новичок» видны заголовок, подсказка и статус каждого шага.
- [ ] Длинные статусы `необязательно`, `после печати` и `после расклейки` не перекрывают название шага.
- [ ] В выборе 1, 2, 4, 6 и 8 макетов на А4 читается пояснение под числом.
- [ ] Итоги `До печати / После печати` помещаются в одну колонку.
- [ ] В карточках офисных сценариев читаются назначение и отметка риска.
- [ ] В компактном списке шаблонов видны описание и badges статуса жизненного цикла.
- [ ] Кнопки режима, мастера и выбора шаблона не обрезаны по вертикали.
- [ ] На ширине рабочего ПК компоновка и размеры подписей не изменились.
- [ ] В предпросмотре и печати А4 нет новых элементов интерфейса.

Критическая проверка для СПН: телефон и QR относятся к печатному макету и должны контролироваться отдельно по печатному чек-листу.

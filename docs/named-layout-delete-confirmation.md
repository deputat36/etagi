# Подтверждение удаления именованного макета

Удаление сохранённого макета требует отдельного явного подтверждения. Обычный клик по кнопке `Удалить выбранный` больше не удаляет запись сразу.

## Диалог

Перед удалением открывается модальный диалог:

- заголовок сообщает, что будет удалён сохранённый макет;
- показывается точное название выбранного макета;
- объясняется, что текущий лист не изменится;
- безопасная кнопка `Оставить макет` получает начальный фокус;
- кнопка `Удалить макет` запускает существующий штатный обработчик удаления;
- Escape равнозначен отмене;
- после закрытия фокус возвращается к кнопке удаления.

Если макет не выбран, диалог не открывается. Сохраняется штатная подсказка `Сначала выберите макет для удаления`.

## Границы

Подтверждение защищает от случайного клика, но само по себе ещё не восстанавливает удалённую запись. Восстановление удалённого именованного макета остаётся следующим отдельным этапом защиты данных.

Не используется `window.confirm`: нативный `<dialog>` доступен для клавиатуры, имеет `aria-labelledby` и `aria-describedby` и не блокирует автоматические проверки браузера.

## Проверка

Контракт входит в общую проверку безопасности хранения:

```bash
npm run validate:storage-safety
npm run smoke:ui-actions
```

Сценарий `Named layout delete confirmation smoke` проверяет:

- штатное поведение без выбранного макета;
- точное название в диалоге;
- начальный фокус на безопасной кнопке;
- отмену кнопкой;
- отмену клавишей Escape;
- удаление только выбранного макета после подтверждения;
- сохранение соседнего макета и текущего листа.

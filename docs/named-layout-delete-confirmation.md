# Подтверждение и восстановление удаления именованного макета

Удаление сохранённого макета требует отдельного явного подтверждения. Обычный клик по кнопке `Удалить выбранный` не удаляет запись сразу.

## Диалог удаления

Перед удалением открывается модальный диалог:

- показывается точное название выбранного макета;
- объясняется, что текущий лист не изменится;
- безопасная кнопка `Оставить макет` получает начальный фокус;
- кнопка `Удалить макет` запускает существующий штатный обработчик удаления;
- Escape равнозначен отмене;
- после закрытия фокус возвращается к кнопке удаления.

Если макет не выбран, диалог не открывается. Сохраняется штатная подсказка `Сначала выберите макет для удаления`.

## Последняя удалённая запись

После подтверждённого успешного удаления отдельный резерв сохраняет последнюю удалённую запись целиком: идентификатор, название, дату обновления и состояние макета. Отмена кнопкой или Escape резерв не создают и не затирают.

Рядом с удалением появляется кнопка `Восстановить «Название»`. Она:

- возвращает запись в список этого браузера;
- не меняет текущий лист;
- выбирает восстановленный макет и переводит фокус к списку;
- удаляет резерв только после успешной записи;
- отключается после использования;
- сохраняет соседние макеты без изменений.

Резерв одноразовый: новое подтверждённое удаление заменяет предыдущую резервную запись.

## Конфликт имени

Если после удаления уже создан новый макет с тем же названием, восстановление не перезаписывает его. Удалённый вариант возвращается под названием `Название (восстановлен)`. При повторном конфликте добавляется следующий свободный номер.

Исходный идентификатор сохраняется, пока он свободен. Если он уже занят, создаётся новый уникальный идентификатор.

## Доступность и ограничения

Не используется `window.confirm`: нативный `<dialog>` имеет `aria-labelledby` и `aria-describedby` и доступен с клавиатуры.

Восстанавливается только последняя удалённая именованная запись. Это отдельный механизм от команды отмены очистки, импорта и загрузки текущего макета.

Фото в именованных макетах, как и до этой доработки, не сохраняются в localStorage; восстановление возвращает тот же облегчённый формат записи.

## Проверка

Контракт входит в общую проверку безопасности хранения:

```bash
npm run validate:storage-safety
npm run smoke:ui-actions
```

Сценарий `Named layout delete confirmation smoke` проверяет полный цикл:

- штатное поведение без выбранного макета;
- точное название и безопасный фокус в диалоге;
- отмену кнопкой и Escape без создания резерва;
- удаление только выбранного макета после подтверждения;
- создание резервной записи только после фактического удаления;
- обычное восстановление и загрузку возвращённого состояния;
- отключение кнопки и удаление использованного резерва;
- конфликт имени без перезаписывания нового макета;
- сохранение соседнего макета и текущего листа.

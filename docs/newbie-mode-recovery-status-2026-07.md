# Статус восстановления режима «Новичок»

Дата: 2026-07

## Контекст

После подключения экспериментального стека режима «Новичок» страница GitHub Pages перестала реагировать на клики. Аварийное отключение этих модулей вернуло работоспособность базового генератора. После этого модули начали возвращаться по одному, с ручной проверкой после каждого шага.

## Рабочая стабильная основа

Текущая входная страница:

```text
https://deputat36.github.io/etagi/
```

Основной файл подключения интерфейсных helper-ов:

```text
assets/js/spnUiMode.js
```

Режим по умолчанию должен оставаться `quick`, чтобы страница открывалась в стабильном варианте.

## Уже возвращено

### 1. CSS-only переключатель «Новичок»

Файл:

```text
assets/js/spnUiMode.js
```

Назначение:

- вернуть кнопку «Новичок»;
- оставить режим «Быстро» как стабильный режим по умолчанию;
- не подключать рискованные helper-ы пачкой.

### 2. Информационный блок новичка

Файл:

```text
assets/js/spnNewbieModeNotice.js
```

Назначение:

- показать пояснение «Безопасный выбор для новичка»;
- не перехватывать клики;
- не менять макет.

### 3. Безопасный фильтр новичка

Файл:

```text
assets/js/spnNewbieMode.js
```

Что изменено:

- поиск по умолчанию заменён на `новичку`;
- обновление идёт через короткий таймер;
- убрано отдельное наблюдение за списком шаблонов и фильтров;
- скрываются рискованные категории и карточки: менеджерские, пустые, нестандартные, «с нуля».

### 4. Печатная подсказка

Файл:

```text
assets/js/spnNewbiePrintGuide.js
```

Назначение:

- подсказать 2 на А4;
- включить линии реза;
- включить безопасные поля;
- показывать прогресс 0/3, 1/3, 2/3, 3/3.

### 5. Поиск шаблонов

Файл:

```text
assets/js/templates.js
```

Что улучшено:

- составные запросы теперь проверяются по словам;
- запрос `офис новичку` может найти шаблон даже при тегах в другом порядке;
- добавлено мягкое совпадение по основе длинных слов.

### 6. Пустое состояние новичка

Файл:

```text
assets/js/spnNewbieEmptyState.js
```

Что изменено:

- модуль переписан без наблюдателей DOM;
- обновляется после действий пользователя;
- показывает блок «Безопасных заготовок не найдено»;
- даёт кнопки `новичку`, `рекомендовано`, `Открыть больше`.

### 7. Офисные фильтры

Файл:

```text
assets/js/spnOfficeTemplateFilters.js
```

Что изменено:

- длинные запросы заменены на короткие ключи;
- `офис новичку` заменён на `новичку`;
- `офис рекомендовано` заменён на `рекомендовано`;
- `объект фото продажа` заменён на `объект`;
- `покупатель покупатели` заменён на `покупател`.

### 8. Пояснение готовности к печати

Файл:

```text
assets/js/spnNewbiePrintGuardNotice.js
```

Назначение:

- показать, какие пункты ещё не готовы;
- дать кнопку «Исправить первое»;
- не блокировать кнопку «Печать / PDF».

### 9. Фото-helper

Файл:

```text
assets/js/spnPhotoLayoutStyle.js
```

Что исправлено перед возвратом:

- убрано наблюдение за изменениями `class`;
- обновление классов поставлено через `requestAnimationFrame`;
- добавлена защита от повторной постановки обновления;
- классы меняются только если действительно отличаются.

### 10. Wizard Flow

Файл:

```text
assets/js/spnWizardFlow.js
```

Что изменено:

- мастер выключен по умолчанию;
- режим пошагового скрытия включается только вручную;
- убран наблюдатель за классами пресетов печати;
- панель мастера можно использовать как навигацию без скрытия блоков.

## Пока не возвращать без отдельного теста

### spnNewbieWizardPatch.js

Причина:

- старый файл содержит дополнительную адаптацию Wizard Flow под новичка;
- попытка переписать файл безопасно была заблокирована connector-ом;
- модуль не критичен для работы генератора.

Решение:

- пока оставить отключённым;
- при необходимости создать новый маленький helper вместо изменения старого.

### spnNewbieFinalCheck.js

Причина:

- старая версия могла нажимать целевые кнопки, включая печать;
- connector заблокировал прямую безопасную перезапись;
- часть задачи уже закрыта через `spnNewbiePrintGuardNotice.js` и `spnNewbiePrintGuide.js`.

Решение:

- пока оставить отключённым;
- не возвращать без ручного теста на GitHub Pages.

### spnNewbiePrintGuard.js

Причина:

- модуль перехватывает кнопку печати;
- после проблемы с зависанием лучше не возвращать блокирующие guard-ы до полного ручного теста;
- вместо него уже подключено только пояснение готовности к печати.

Решение:

- оставить отключённым;
- если понадобится защита от преждевременной печати, делать её отдельным мягким предупреждением, а не блокировкой.

## Ручной тест после каждого шага

Проверять через `Ctrl + F5` или инкогнито:

1. Страница открывается.
2. Кнопки нажимаются.
3. Переключаются режимы `Новичок`, `Быстро`, `Расширенно`.
4. Работают категории шаблонов.
5. В режиме `Новичок` не видны менеджерские и пустые заготовки.
6. Работает выбор 1, 2, 4, 6, 8 на А4.
7. Работают линии реза и безопасные поля.
8. Работает кнопка «Печать / PDF».
9. Переключение `Пошагово` не зависает и режим `Все блоки` возвращает полный интерфейс.

## Главный принцип дальше

Не возвращать экспериментальные модули пачкой. Каждый следующий helper должен быть:

- без постоянного наблюдения за DOM;
- без циклов изменения классов;
- без автоматического клика по печати;
- без блокировки основных кнопок;
- с ручной проверкой на GitHub Pages.

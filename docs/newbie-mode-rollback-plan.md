# План аварийного отката режима «Новичок»

Документ нужен на случай, если после очередного изменения интерфейс снова зависнет, перестанут нажиматься кнопки или страница начнёт открываться с ошибкой.

## Главный принцип

Откат выполняется не удалением файлов, а временным отключением импортов в `assets/js/spnUiMode.js`. Так проще вернуть модуль после исправления и не потерять историю работы.

## Что проверять сначала

1. Открыть сайт в режиме инкогнито.
2. Выполнить жёсткую перезагрузку через `Ctrl + F5`.
3. Проверить, что `index.html` подключает актуальный `spnUiMode.js` с cache-busting параметром.
4. Открыть консоль браузера и посмотреть первую ошибку JavaScript.
5. Если интерфейс зависает без явной ошибки, отключать последние подключённые helper-модули по одному.

## Безопасная базовая конфигурация

В `assets/js/spnUiMode.js` можно оставить основные стабильные модули:

```js
import './spnTearOffEditor.js';
import './spnBlockControls.js';
import './spnQrStandaloneStyle.js';
import './spnContactEditor.js';
import './spnBrandEditor.js';
import './spnQrEditor.js';
import './spnPriceHelper.js';
import './spnParamsHelper.js';
import './spnAreaHelper.js';
import './spnPhoneHelper.js';
```

После такого отката должны работать режимы `Быстро` и `Расширенно`, выбор задачи, выбор шаблона, редактирование данных, проверка и печать.

## Порядок отключения модулей новичка

Если проблема появилась после возврата режима `Новичок`, отключать модули лучше в таком порядке:

1. `spnNewbieWizardPatch.js` — управляет поведением мастера в режиме новичка.
2. `spnWizardFlow.js` — добавляет мастер шагов и режим `Пошагово`.
3. `spnPhotoLayoutStyle.js` — добавляет классы фото-раскладки.
4. `spnNewbiePrintGuard.js` — блокирует преждевременную печать в режиме новичка.
5. `spnNewbiePrintGuardNotice.js` — показывает notice готовности к печати.
6. `spnNewbieFinalCheck.js` — добавляет финальный чек-лист перед печатью.
7. `spnNewbiePrintGuide.js` — добавляет подсказки печатных настроек.
8. `spnNewbieEmptyState.js` — показывает пустое состояние при отсутствии подходящих шаблонов.
9. `spnNewbieMode.js` и `spnNewbieModeNotice.js` — базовый фильтр и notice режима новичка.

После каждого отключения нужно обновлять cache-busting в `index.html` и проверять сайт через `Ctrl + F5`.

## Как понять, что виноват конкретный модуль

Модуль считается подозрительным, если после его отключения:

- страница снова открывается;
- кнопки начинают нажиматься;
- переключение режимов работает;
- печать открывается в режимах `Быстро` и `Расширенно`;
- в консоли исчезает ошибка, связанная с этим модулем.

## Что нельзя делать при откате

- Не удалять helper-файлы без необходимости.
- Не отключать все скрипты в `index.html`, если достаточно отключить импорт в `spnUiMode.js`.
- Не оставлять старый cache-busting после изменения импортов.
- Не подключать обратно сразу несколько отключённых модулей.
- Не возвращать глобальный перехват всех кликов документа для защиты печати.
- Не подключать модуль, который сам меняет классы и одновременно наблюдает изменение этих же классов.

## Минимальная проверка после отката

1. Сайт открывается.
2. Кнопки `Новичок`, `Быстро`, `Расширенно` нажимаются.
3. В режиме `Быстро` можно выбрать шаблон.
4. Можно заполнить телефон.
5. Работает кнопка `Проверить`.
6. Работает кнопка `Печать / PDF`.
7. Нет бесконечных изменений в интерфейсе.

## Как возвращать модуль после исправления

1. Исправить модуль отдельным коммитом.
2. Подключить только один импорт.
3. Обновить cache-busting в `index.html`.
4. Проверить сайт через `Ctrl + F5`.
5. Пройти `docs/newbie-mode-regression-checklist.md`.
6. Зафиксировать шаг в issue по Wizard Flow.

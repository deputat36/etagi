# Единый контроль форматов A4

Официальные форматы генератора:

```text
1, 2, 3, 4, 6 и 8 макетов на A4
```

Для каждого значения выполняется один browser-сценарий с одинаковыми синтетическими контактами и коротким текстом.

## Общие проверки

Каждый формат обязан сохранить:

- точное количество карточек;
- полный телефон `+7 960 125-18-77`;
- имя специалиста `Анна Качкина`;
- контактный блок внутри карточки;
- карточку внутри листа A4;
- отсутствие класса `overflow`;
- минимальный размер телефона для своей плотности.

Минимальные размеры телефона:

| Формат | Минимум |
|---|---:|
| 1 на A4 | 30 px |
| 2 на A4 | 28 px |
| 3 на A4 | 24 px |
| 4 на A4 | 20 px |
| 6 на A4 | 18 px |
| 8 на A4 | 16 px |

## Политика отрывных полос

Для форматов 1, 2, 3 и 4 отрывные полосы доступны. Browser-сценарий включает их и требует:

- ровно восемь полос в каждой карточке;
- полный телефон в каждой полосе;
- общий блок и каждая отдельная полоса внутри карточки;
- отсутствие overflow после включения полос.

Форматы 6 и 8 считаются плотными. Для них система обязана:

- автоматически снять отметку отрывных полос;
- заблокировать checkbox;
- не создавать `.tears` в карточках;
- сохранить телефон, имя СПН и контактный блок.

## Запуск

Статический контракт:

```bash
node tools/validate-print-format-coverage.mjs
```

Browser-сценарий:

```bash
node tools/run-print-format-coverage-smoke.mjs
```

GitHub Actions выполняет их последовательно в workflow `Validate print format coverage`. При ошибке создаётся artifact `print-format-coverage-failure` с последним форматом и снимком состояния.

Эта автоматическая проверка не заменяет физическую печать issue #40.

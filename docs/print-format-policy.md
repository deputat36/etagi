# Политика форматов A4

Официальный набор форматов генератора:

```text
1, 2, 3, 4, 6 и 8 макетов на листе A4
```

Этот список является единым для:

- кнопок выбора количества в интерфейсе;
- нового и старого JSON-макета;
- автосохранения и локально сохранённых макетов;
- сетки предпросмотра и печатного renderer;
- JSON Schema `data/layout-file.schema.json`.

## Неподдерживаемые значения

Значения 10 и 12 не входят в официальный набор. Они могли отрисовываться старым renderer при ручном изменении JSON или наличии старого локального состояния, но не имели кнопок интерфейса и отдельной печатной проверки.

Теперь:

- импорт JSON с 10 или 12 отклоняется до замены текущего макета;
- сообщение перечисляет допустимые форматы;
- выбранный файл сбрасывается и может быть выбран повторно;
- старый autosave, последний или именованный макет с 10/12 безопасно открывается как 2 на A4;
- renderer дополнительно нормализует неподдерживаемое состояние до 2 и не содержит отдельных сеток 2×5 или 2×6.

## Уровень подтверждения

Форматы 1, 2, 3, 4, 6 и 8 официально доступны в продукте и являются допустимыми значениями состояния.

Это не означает, что все они уже считаются проверенными физической печатью. На текущем этапе:

- автоматические screenshot-сценарии покрывают 1, 2, 3, 4, 6 и 8 макетов на A4;
- сценарий 3 на A4 проверяет три полосы, телефон, контакты, overflow и восемь отрывных полос в каждой карточке;
- сценарий 6 на A4 проверяет шесть карточек 2×3, экономный режим, отсутствие отрывных полос, читаемость заголовка и телефона, контакты и overflow;
- отдельный сценарий 8 на A4 проверяет восемь карточек 2×4, экономный режим, отсутствие фото и отрывных полос, границы карточек и контактов;
- для формата 8 установлен минимальный размер телефона 16 px и минимальный размер заголовка 12 px;
- проверка на рабочем ПК, телефоне и офисном принтере остаётся в issue #40;
- выпуск 3.86.0 остаётся заблокирован до ручных подтверждений.

## Контроль

Статическая проверка:

```bash
npm run validate:layout-file
npm run validate:print-screenshots
node tools/validate-eight-print-screenshot.mjs
```

Browser regression входит в:

```bash
npm run smoke:ui-actions
```

Основная печатная screenshot-матрица запускается командой:

```bash
npm run screenshots:print
```

Отдельная проверка восьми макетов запускается командой:

```bash
node tools/run-eight-print-screenshot.mjs
```

Сценарий `Print format policy smoke` проверяет точный набор кнопок, нормализацию старого локального состояния, отказ импорта 10/12 без изменения текущего макета и защитный fallback renderer. Сценарии `three-tearoffs`, `six-economy` и отдельный `eight-economy` подтверждают геометрию и контакты плотных форматов 3, 6 и 8 на A4.

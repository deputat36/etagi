# 3.3.0 — проверка ссылок HTML-страниц

Усилена проверка статических связей после добавления пользовательских страниц помощи.

Что изменилось:

- `tools/validate-assets.mjs` теперь проверяет не только `index.html`, но и все HTML-страницы проекта;
- проверяются внутренние ссылки `<a href>`, стили `<link href>`, скрипты `<script src>` и изображения `<img src>`;
- относительные ссылки проверяются от папки конкретной HTML-страницы;
- workflow теперь запускается и при изменении `help/**`;
- версия `package.json` обновлена до `3.3.0`;
- версия состояния приложения обновлена до `3.3.0`.

Зачем это нужно:

В проекте появились отдельные страницы помощи. Теперь если ссылка из генератора или страницы помощи будет вести на несуществующий файл, проверка покажет ошибку до публикации на GitHub Pages.

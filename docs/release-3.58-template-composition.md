# 3.58.0 — шаблон применяет свою композицию

Исправлена важная проблема генерации макетов: выбранный шаблон теперь применяет свою композицию, а не наследует скрытую структуру от предыдущего макета.

Изменены файлы:

- `assets/js/app.js`;
- `package.json`;
- `assets/js/state.js`.

Что было раньше:

- при выборе шаблона сохранялась старая видимость блоков;
- сохранялся старый порядок блоков;
- из-за этого подъездный, объектный, доверительный или фото-шаблон могли применяться не полностью;
- пользователь выбирал один шаблон, но получал гибрид старого макета и нового текста.

Что изменено:

- при выборе шаблона создаётся чистая база из `cloneDefaultState()`;
- поверх неё применяются данные выбранного шаблона;
- профиль СПН — имя и телефон — сохраняется отдельно;
- загруженные фото сохраняются отдельно;
- порядок блоков берётся из шаблона, если он есть, иначе используется базовый порядок;
- старые настройки видимости и порядка больше не переносятся автоматически;
- фильтр `Подъезд` больше не цепляет все шаблоны со словом `куплю`.

Дополнительно:

- добавлено действие для замечания `shortQr`, чтобы пользователь мог перейти к полю QR-ссылки;
- версия проекта обновлена до `3.58.0`.

Зачем это нужно:

Шаблон должен быть не просто набором текста, а готовой логикой макета. Теперь выбранный шаблон действительно задаёт структуру макета, а профиль специалиста остаётся неизменным.

# Релиз-кандидат 3.86.0

Статус: DRAFT

Целевая версия: 3.86.0

Текущая опубликованная версия: 3.85.0

Причина статуса `DRAFT`: автоматические проверки пройдены, но ручная проверка доступа к печати на рабочем ПК и телефоне, физическая печатная приёмка и менеджерская проверка чувствительных шаблонов ещё не подтверждены.

## Что войдёт в 3.86.0

### Рабочий путь СПН

- режимы `Новичок`, `Быстро` и `Расширенно`;
- пошаговый Wizard Flow с компактными статусами восьми шагов;
- закреплённая панель предпросмотра и быстрый переход к штатной проверке печати;
- сохранение профиля, макетов и полный локальный backup;
- мобильный режим исполнителя расклейки;
- рабочее пространство `После печати` с заданием, отчётом и аналитикой.

### Шаблоны и безопасность

- расширенная библиотека сценариев Борисоглебска;
- office-уровни, риски и заметки менеджера;
- жизненный цикл `working / test / deprecated`;
- уникальные итоговые `templateId` и безопасные алиасы исторических дублей;
- полностью размеченные пакеты `templates_sales.json`, `templates_trust.json`, `templates.json` и `templates_ab_tests.json`;
- базовые прямые заявления о спросе переведены в `test / manager / high`, безопасная оценка цены оставлена новичку;
- custom-конструкторы доступны только как manager / not recommended;
- контролируемые ипотечные, юридические, частные и A/B-сценарии;
- менеджерский notice для чувствительных формулировок;
- обязательный бланк `docs/manager-sensitive-template-review-3.86.0.md` для решения по чувствительным сценариям;
- доказательный пакет `docs/manager-sensitive-template-evidence-3.86.0.md` с текущими текстами, статусами, рисками и managerNote для 12 автоматически обнаруженных шаблонов.

### Фото и печать

- режимы `photo_left`, `photo_card`, `newbuild_visual` и `agent_brand_photo`;
- безопасное отображение планировок и портретов без обрезки;
- ограничения фото-компоновок до 1–2 макетов на А4;
- синхронные классы компоновки при рендере;
- экономный цвет для плотной печати без чрезмерных сплошных заливок;
- сохранение фотографий, QR, логотипа и фирменного акцента.

### Контроль и аналитика

- проверки качества и безопасные действия исправления;
- защита печати новичка;
- история отчётов и CSV;
- устойчивые связки `templateId + место`;
- сводка менеджеру за 7 дней, 30 дней и всю историю;
- удаление ошибочного отчёта с подтверждением.

### Надёжность разработки

- единый Promise-кеш шаблонов;
- единый asset-version из `package.json`;
- команда `npm run assets:stamp` и проверка `validate:asset-version`;
- release-candidate gate, запрещающий преждевременно выставить версию 3.86.0 без viewport-, физической и менеджерской приёмки;
- автоматическая проверка свежести `docs/template-portfolio-inventory.generated.md`;
- `validate:release-candidate` запускает `node tools/validate-manager-sensitive-review.mjs` и не допускает расхождения основного бланка, evidence-пакета, исходных текстов и office-политики;
- `npm run release:status` объединяет версии, CI, ручные чекбоксы и следующий допустимый шаг без изменения файлов;
- общий `npm run validate`;
- настоящий browser smoke через Chrome DevTools Protocol;
- пять независимых печатных screenshot-сценариев через Chrome DevTools Protocol;
- диагностические artifacts для статических, браузерных и печатных ошибок.

## Автоматические доказательства

Последний полный контроль: GitHub Actions workflow run #1474 от 14 июля 2026 года.

Подтверждено этим зафиксированным запуском:

- `validate` — успешно, включая `validate:asset-version`, `validate:release-candidate`, office-политику и CDP-контракты;
- `browser-smoke` — успешно;
- `print-screenshot` для пяти сценариев — успешно;
- `collect-print-screenshots` — успешно;
- итоговый artifact `print-screenshots` содержит пять PNG, пять JSON-метафайлов и `manifest.json`;
- плотный сценарий `four-contacts` использует светлую контактную зону и цветную рамку;
- 31 CSS/JS entry-ресурс использует единый токен `?v=3.85.0`;
- старый ручной asset-token отсутствует;
- browser-smoke ждёт фактический `data-status=passed/failed` через `Runtime.evaluate` без `virtual-time-budget` и `dump-dom`;
- ошибка harness не повторяется и не может быть замаскирована второй попыткой Chrome.

Run #1458 корректно обнаружил пропущенный чувствительный шаблон `seller_empty_flat`. После исправления финальный run #1474 подтвердил усиленный release gate; `one-showcase` успешно завершился после точечного rerun подтверждённо нестабильного job.

Автоматическая техническая часть не заменяет ручной сценарий 0, физическую печать, проверку QR камерой телефона и менеджерскую оценку чувствительных рекламных формулировок.

## Порядок перевода в READY

1. Выполнить сценарий 0 на рабочем ПК и реальном телефоне.
2. Заполнить ручную печатную приёмку.
3. Сверить доказательный пакет, заполнить менеджерскую проверку чувствительных шаблонов и закрыть issue #51.
4. Выполнить `node tools/validate-manager-sensitive-review.mjs`.
5. Проверить сводку командой `npm run release:status -- --strict`: до закрытия блокеров ожидается код `2`, после согласованного перехода в `READY` — код `0`.
6. Изменить `Статус: DRAFT` на `Статус: READY`.
7. Обновить `package.json` до `3.86.0`.
8. Выполнить `npm run assets:stamp`.
9. Добавить раздел `## 3.86.0` в `docs/changelog.md`.
10. Запустить полный `npm run validate`.
11. Подтвердить `browser-smoke` и всю screenshot matrix.
12. Только после этого создавать релиз и обновлять опубликованную версию.

До выполнения этих шагов повышение версии и финальный changelog считаются преждевременными.

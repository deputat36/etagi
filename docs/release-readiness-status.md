# Сводный статус готовности релиза

Команда собирает состояние релиз-кандидата 3.86.0 из существующих файлов и ничего не изменяет:

```bash
npm run release:status
```

Она показывает:

- текущую и целевую версии;
- статус `DRAFT / READY`;
- номер зафиксированного полного CI-run;
- количество выполненных и оставшихся пунктов релизного бланка;
- остаток ручной печатной приёмки issue #40;
- остаток менеджерской проверки issue #51;
- наличие финального раздела версии в changelog;
- следующий допустимый шаг.

Если незакрытые пункты остаются, компактный вывод подсказывает команду для полного перечня.

## Подробный перечень

Для работы на планёрке, печати или последовательного прохождения ручной приёмки используется:

```bash
npm run release:status -- --details
```

Режим выводит три пронумерованные группы:

- блокирующие пункты релизного бланка;
- все незакрытые проверки issue #40;
- все незакрытые решения issue #51.

Тексты извлекаются напрямую из действующих Markdown-бланков. Отчёт проверяет, что число извлечённых строк совпадает со счётчиками чекбоксов. Поэтому добавленный или удалённый пункт не должен незаметно исчезнуть из подробного списка.

## JSON

Для автоматического чтения используется:

```bash
npm run release:status -- --json
```

JSON содержит версии, статус, сводки чекбоксов, массивы `pendingItems`, blockers, ошибки согласованности и поле `nextAction`.

## Strict-режим

Перед фактическим выпуском можно выполнить:

```bash
npm run release:status -- --strict
```

Режим завершится:

- кодом `0`, если пакет согласован и действительно готов к выпуску;
- кодом `1`, если документы или версии рассинхронизированы;
- кодом `2`, если состояние согласовано, но релиз ещё заблокирован ручными условиями.

Подробный список совместим со strict-проверкой:

```bash
npm run release:status -- --details --strict
```

Для текущего `DRAFT` код `2` является ожидаемым: issue #40 и issue #51 не пройдены, версия остаётся 3.85.0, финальный changelog 3.86.0 отсутствует.

## Автоматическая проверка

GitHub Actions после общего `npm run validate` отдельно выполняет:

```bash
npm run test:release-status-details
```

Тест сверяет JSON и подробный вывод, количество строк во всех трёх группах, компактную подсказку и совместимость `--details --strict`.

## Ограничения

Отчёт не заменяет физическую печать, проверку QR двумя телефонами и решение менеджера по чувствительным шаблонам. Он только объединяет уже зафиксированное состояние и не изменяет файлы, issues или статус релиза.

# История отчётов после расклейки

Документ для менеджера и СПН.

## Зачем нужна история

Один отчёт показывает результат одной расклейки. История отчётов помогает понять, какие связки работают лучше:

```text
макет → место → количество листов → отклики → целевые обращения → следующий шаг
```

## Что сохраняется

В историю сохраняются:

- дата;
- место расклейки;
- заголовок макета;
- количество листов;
- примерное количество объявлений;
- звонки и сообщения;
- целевые обращения;
- примерный процент отклика;
- автоматический вывод по результату;
- следующий шаг;
- заметки.

История хранится локально в браузере пользователя и не отправляется на сервер.

## Как пользоваться

1. После расклейки заполнить блок «Отчёт после расклейки».
2. Посмотреть автоматический вывод под метриками.
3. Нажать «В историю».
4. При необходимости нажать «CSV» и выгрузить историю.
5. Если расклейку нужно повторить, нажать «Повторить» у нужной записи.

## Автоматический вывод

Генератор подсказывает следующий смысловой шаг:

- если откликов нет — проверить место, заголовок и читаемость телефона;
- если отклики есть, но целевых нет — уточнить текст и сделать предложение конкретнее;
- если целевые обращения есть — повторить тест или закрепить связку;
- если результат хороший — повторить в похожих домах и добавить шаблон в рекомендованные.

## Как читать результаты

Если откликов нет:

- изменить место;
- сократить текст;
- усилить заголовок;
- попробовать другой формат печати.

Если звонки есть, но целевых мало:

- уточнить аудиторию;
- добавить конкретику по дому, району, цене или типу объекта;
- убрать слишком широкие обещания.

Если целевые обращения есть:

- повторить макет в похожих домах;
- сохранить связку как рабочую;
- добавить шаблон в рекомендованные.

## Ограничения

История хранится в `localStorage`. Если пользователь очистит данные браузера, история пропадёт. Для офисного контроля лучше периодически выгружать CSV.

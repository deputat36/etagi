# Runtime registry модулей

Реестр нужен для безопасного перехода от большого связанного runtime к контролируемой архитектуре. Он не меняет порядок загрузки и не отключает функции, а фиксирует фактический граф модулей, который загружается основным `index.html`.

## Файлы

```text
data/runtime-modules.json
 tools/build-runtime-module-registry.mjs
 tools/validate-runtime-module-registry.mjs
```

`data/runtime-modules.json` содержит:

- все локальные `<script type="module">` из `index.html`;
- все достижимые статические, side-effect и dynamic imports;
- обратные связи `importedBy`;
- назначение каждой точки входа;
- домен модуля;
- итоговое количество точек входа, модулей и связей.

## Обновление

После добавления, удаления или переноса runtime-модуля выполнить:

```bash
npm run runtime:registry
npm run validate:runtime-modules
```

Общий `npm run validate` также запускает проверку реестра.

## Что запрещает validator

- отсутствующий или некорректный JSON;
- пропущенную точку входа `index.html`;
- новый импорт без обновления реестра;
- запись о несуществующем файле;
- повторяющиеся module paths;
- несогласованные связи `imports` и `importedBy`;
- неполный side-effect bundle `spnUiMode.js`;
- query/hash в каноническом пути модуля.

## Границы текущего этапа

Реестр пока не утверждает единственного владельца DOM, storage keys, observers или style IDs. Эти данные будут добавлены отдельной следующей итерацией, чтобы не смешивать инвентаризацию графа с архитектурными решениями.

Реестр также не является основанием для массового объединения или удаления модулей. Любой рефакторинг выполняется отдельным небольшим PR с browser smoke и профильными проверками.

# Как добавлять шаблоны расклеек

Документ для безопасного расширения библиотеки шаблонов.

## Где лежат шаблоны

Шаблоны хранятся в JSON-файлах в папке `data`.

Сейчас используются пакеты:

```text
 data/templates.json
 data/templates_custom.json
 data/templates_sales.json
 data/templates_extra.json
 data/templates_borisoglebsk.json
 data/templates_tellerman_sad.json
 data/templates_entrance.json
 data/templates_ab_tests.json
 data/templates_trust.json
```

Подключение файлов находится в `assets/js/templates.js`.

## Когда создавать отдельный файл

Лучше не складывать всё в один большой JSON. Новый файл нужен, если появляется отдельная логика:

```text
 data/templates_<город>.json
 data/templates_<сценарий>.json
 data/templates_<кампания>.json
```

Примеры:

```text
 data/templates_borisoglebsk.json
 data/templates_entrance.json
 data/templates_tellerman_sad.json
```

После создания нового файла его нужно добавить в массив `TEMPLATE_FILES` в `assets/js/templates.js`.

## Минимальная структура шаблона

```json
{
  "id": "seller_example",
  "goal": "seller",
  "title": "Название шаблона",
  "note": "Короткое пояснение, когда использовать.",
  "tags": ["офис", "рекомендовано", "новичку", "собственники", "подъезд"],
  "office": {
    "recommended": true,
    "level": "newbie",
    "scenario": "entrance_owner_buy",
    "risk": "low",
    "recommendedPrintCount": 4,
    "managerNote": "Подходит новичку: проверить телефон и район перед печатью."
  },
  "density": "normal",
  "photo": "none",
  "printCount": 2,
  "data": {
    "agentName": "",
    "agentPhone": "",
    "area": "",
    "propertyType": "квартира",
    "price": "",
    "params": "",
    "headline": "КРУПНЫЙ\nЗАГОЛОВОК",
    "description": "Основной текст объявления.",
    "benefits": "Преимущество 1\nПреимущество 2\nПреимущество 3",
    "qrCaption": "",
    "qrLink": "",
    "showBrand": true,
    "showBenefits": true,
    "showMeta": true,
    "tearOffs": true,
    "colorMode": "brand",
    "splitMode": "auto",
    "headlineScale": 1,
    "phoneScale": 1.25,
    "pageMargin": 7,
    "pageGap": 4,
    "flyerPadding": 5
  }
}
```

## Поля верхнего уровня

`id` — уникальный идентификатор латиницей. Лучше использовать формат: `цель_сценарий_уточнение`, например `seller_empty_flat`.

`goal` — задача шаблона:

```text
seller
buyer
object
newbuild
service
rent
brand
private
all
```

`title` — название карточки шаблона в интерфейсе.

`note` — короткая подсказка для СПН.

`tags` — слова для поиска, сценарных фильтров и офисных подборок.

`office` — структурированные офисные признаки. Они не заменяют `tags`, а дополняют их: помогают понять, кому можно дать шаблон, какой риск у макета и какой формат печати лучше выбрать.

`density` — плотность:

```text
airy     свободный макет
normal   обычный макет
dense    плотный макет
```

`photo` — режим фото:

```text
none
one
two
plan
```

`printCount` — сколько макетов на А4. Обычно 1, 2 или 4.

## Офисные теги

Офисные фильтры работают через обычный поиск по тегам. Чтобы шаблон попадал в нужную подборку, добавьте соответствующие слова в `tags`.

```text
офис             общий офисный шаблон
рекомендовано    безопасный шаблон для регулярной работы
новичку          подходит для первых макетов СПН
менеджер         лучше проверить перед массовой печатью
подъезд          подъездная расклейка
Теллерманов сад  локальная подборка по ЖК
```

Пример:

```json
"tags": ["офис", "рекомендовано", "новичку", "подъезд", "собственники"]
```

Если шаблон связан с ипотекой, новостройкой, юридической темой, объектом с фото/QR или нестандартным обещанием, лучше добавить тег `менеджер`.

## Office-метаданные

`office` — это прототип более явной офисной разметки. Он нужен, чтобы в будущем не полагаться только на текстовые теги и поиск.

Пример:

```json
"office": {
  "recommended": true,
  "level": "newbie",
  "scenario": "entrance_owner_buy",
  "risk": "low",
  "recommendedPrintCount": 4,
  "managerNote": "Подходит новичку: проверить телефон и район перед печатью."
}
```

Поля:

```text
recommended             true / false, можно ли регулярно использовать в офисе
level                   newbie / experienced / manager
scenario                внутренний сценарий латиницей
risk                    low / medium / high
recommendedPrintCount   рекомендованное количество макетов на А4
managerNote             короткая заметка, что проверить перед печатью
```

Правила:

- `level: "newbie"` требует тег `новичку`;
- `level: "manager"` требует тег `менеджер`;
- `recommended: true` требует тег `рекомендовано`;
- `scenario` должен быть латиницей, цифрами, `_` или `-`;
- `managerNote` должен объяснять, что именно проверить.

Сейчас `office`-метаданные обязательны для `data/templates_entrance.json`. В остальных пакетах они будут внедряться постепенно.

## Автоматическая проверка офисных тегов

Офисная разметка проверяется командой:

```bash
npm run validate:templates
```

Сейчас строгие правила включены для трёх ключевых пакетов:

```text
data/templates_entrance.json
data/templates_borisoglebsk.json
data/templates_tellerman_sad.json
```

Для `data/templates_entrance.json` обязательны:

```text
офис
рекомендовано
подъезд
новичку или менеджер
office-метаданные
```

Для `data/templates_borisoglebsk.json` обязательны теги:

```text
офис
рекомендовано
Борисоглебск
```

Исключения:

- объектные шаблоны с фото/QR должны иметь `менеджер` и не должны иметь `новичку`;
- частные шаблоны без бренда должны иметь `менеджер` и не должны иметь `новичку`.

Для `data/templates_tellerman_sad.json` обязательны теги:

```text
офис
Борисоглебск
Теллерманов сад
новичку или менеджер
```

Если новый шаблон не проходит эти правила, лучше не обходить валидатор, а решить: шаблон безопасен для новичка или его должен проверить менеджер.

## Текстовые поля

`headline` — главный заголовок. Можно переносить строки через `\n`.

Хорошо:

```text
КУПЛЮ\nКВАРТИРУ\nВ ЭТОМ ДОМЕ
```

Плохо:

```text
Здравствуйте, я специалист по недвижимости и хочу купить квартиру в вашем доме
```

`description` — основной текст. Лучше 1–2 коротких предложения.

`benefits` — преимущества, каждое с новой строки.

Оптимально 3–4 строки. Если строк больше 6, текст может не поместиться.

## Настройки фирменности

Для фирменного макета:

```json
"showBrand": true,
"colorMode": "brand"
```

Для частного объявления:

```json
"showBrand": false,
"colorMode": "private"
```

## QR и отрывные телефоны

Для подъездных объявлений обычно лучше включать отрывные телефоны:

```json
"tearOffs": true
```

QR лучше использовать для крупных макетов на 1–2 объявления на А4. Для плотных макетов на 4 объявления QR может быть слишком мелким.

## Проверка перед публикацией

После добавления или изменения шаблонов запустить:

```bash
npm run validate
```

Проверка покажет:

- ошибки JSON;
- отсутствующие обязательные поля;
- неизвестные значения `goal`, `density`, `photo`, `colorMode`, `splitMode`;
- ошибки офисных тегов;
- ошибки `office`-метаданных;
- длинные заголовки и описания;
- дубли `id`;
- проблемы с подключением файлов;
- ошибки офисной разметки для ключевых пакетов шаблонов.

## Правило качества

Хороший шаблон должен отвечать на три вопроса:

1. К кому обращаемся?
2. Что хотим получить от человека?
3. Почему ему не страшно позвонить?

# Актуальность инвентаризации шаблонов

## Проблема

Файл `docs/template-portfolio-inventory.generated.md` является снимком библиотеки шаблонов. После изменения office-метаданных, жизненного цикла, алиасов или пакетов он может устареть, даже если сами шаблоны и валидаторы корректны.

## Проверка

Актуальность отчёта автоматически проверяется внутри:

```bash
npm run validate:template-portfolio
```

Для отдельной диагностики можно запустить:

```bash
node tools/validate-template-inventory-freshness.mjs
```

Проверка временно запускает существующий генератор:

```bash
npm run templates:inventory
```

Затем она:

1. читает новый отчёт;
2. восстанавливает исходный файл;
3. сравнивает содержимое без учёта динамической строки времени;
4. завершает проверку ошибкой, если цифры, таблицы или списки отличаются.

Рабочее дерево после проверки не должно изменяться.

## Если отчёт устарел

Запустить:

```bash
npm run templates:inventory
```

Проверить изменения и сохранить обновлённый `docs/template-portfolio-inventory.generated.md`.

В GitHub Actions новый отчёт выводится между маркерами:

```text
GENERATED_TEMPLATE_INVENTORY_BEGIN
GENERATED_TEMPLATE_INVENTORY_END
```

Поэтому актуальную версию можно получить из artifact `validation-failure`, даже если локальная среда не может клонировать репозиторий.

## Что считается регрессией

- отчёт показывает старое office-покрытие;
- в отчёте отсутствуют новые тестовые или устаревшие шаблоны;
- изменились алиасы ID, но отчёт не обновлён;
- проверка оставляет изменённый файл после завершения;
- сравнение игнорирует что-либо кроме динамической строки времени.

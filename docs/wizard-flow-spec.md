# Wizard Flow v1

## Цель

Упростить создание макета расклейки. Пользователь должен идти по шагам, а не видеть все настройки сразу.

## Продуктовый принцип

```text
Новичок видит следующий нужный шаг.
Опытный СПН может раскрыть дополнительные настройки.
Менеджер видит проверку, риски и результат.
```

## Целевой порядок работы

```text
1. Цель
2. Количество объявлений на А4
3. Категория заготовок
4. Выбор макета
5. Редактирование текста
6. Фото и QR
7. Проверка
8. Печать / PDF
9. Задание на расклейку
10. Отчёт по результату
```

## Текущая реализация в коде

Сейчас `assets/js/spnWizardFlow.js` реализует 8 экранных шагов. Некоторые целевые пункты объединены, чтобы не переписывать ядро приложения резко.

```text
1. Цель       → цель + рабочая ситуация + выбор количества на А4
2. Заготовка  → категории заготовок + список шаблонов
3. Текст      → контакты, объект, заголовок, описание, преимущества
4. Фото / QR  → фото, режим фото, ссылка и подпись QR
5. Проверка   → настройки печати + контроль качества
6. Сохранить  → сохранение текущего и именованного макета
7. Задание    → задание на расклейку
8. Отчёт      → отчёт после расклейки, история, повтор и CSV
```

Такой формат безопасен для текущей архитектуры: мастер управляет видимостью крупных секций, но не ломает `app.js` и не меняет хранение состояния.

## Шаг 1. Цель и количество на А4

Показывать крупные карточки целей:

- найти собственника;
- найти покупателя;
- продать объект;
- новостройка;
- оценка / услуга;
- аренда;
- личный бренд;
- частное объявление.

Рядом или сразу под целью показывать количество объявлений на А4:

```text
1 крупно
2 на А4 — чаще всего
4 экономно
6 мелко
8 мини
```

По умолчанию должен быть выбран вариант `2 на А4`.

Подсказки:

- `1 крупно` — объект, фото, витрина, объявление на стойку;
- `2 на А4` — основной рабочий формат для офиса;
- `4 экономно` — подъезды, короткие тексты, отрывные телефоны;
- `6 мелко` и `8 мини` — только очень короткий текст без перегруза.

## Шаг 2. Заготовка

Этот шаг объединяет выбор категории и выбор конкретного шаблона.

Категории уже оформлены как карточки решений. Пользователь выбирает рабочий сценарий, а не технический поисковый фильтр.

Категории:

- рекомендовано;
- новичку;
- менеджер;
- подъезд;
- собственники;
- покупатели;
- объекты;
- новостройки;
- Теллерманов сад;
- доверие;
- с фото;
- пустые под проверку менеджера.

Карточки шаблонов показывают бейджи и пояснения. Если у шаблона есть `office`-метаданные, интерфейс использует:

```text
office.level
office.risk
office.recommendedPrintCount
office.managerNote
```

Это позволяет объяснять, почему шаблон подходит новичку, почему нужен менеджер, какой риск у макета и какой формат А4 лучше использовать.

## Шаг 3. Редактирование текста

Каждый видимый текст в макете должен быть редактируемым.

Основные группы полей:

- контакты;
- объект / район;
- текст объявления;
- дополнительный блок;
- состав макета;
- порядок блоков.

В режиме `Новичок` нужно постепенно скрывать сложные настройки и показывать только то, что обязательно для выбранного шаблона.

## Шаг 4. Фото и QR

Показывать фото и QR только когда они нужны выбранному макету или пользователь сам включил эти блоки.

Для фото нужны отдельные компоновки:

- крупное фото сверху;
- фото слева;
- карточка объекта с фото;
- планировка;
- изображение ЖК;
- фото СПН для личного бренда.

Фото не должно выглядеть узкой полоской.

## Шаг 5. Проверка и печать

Текущая реализация объединяет печатные настройки и контроль качества в один шаг.

На шаге должны быть видны:

- количество объявлений на листе;
- деление листа;
- линии реза;
- безопасные поля;
- отрывные телефоны;
- кнопка `Проверить`;
- список ошибок, важных замечаний и советов;
- кнопка `Печать / PDF`.

Следующее улучшение: добавить итоговый вердикт качества:

```text
Можно печатать.
Нужно исправить телефон.
Лучше показать менеджеру.
Макет перегружен — выберите 2 на А4 или сократите текст.
```

## Шаг 6. Сохранить

Показывать:

- название макета;
- сохранение именованного макета;
- загрузку сохранённого макета;
- удаление сохранённого макета;
- сохранение последнего макета;
- открытие и скачивание JSON.

В режиме `Новичок` этот шаг не должен отвлекать до печати. Его можно показывать после проверки или после печати.

## Шаг 7. Задание на расклейку

Шаг связан с блоком:

```text
#spnDistributionTask
```

Показывать:

- количество листов;
- место расклейки;
- ответственного;
- срок;
- текст задания;
- кнопку копирования.

Задание должно автоматически учитывать:

- заголовок макета;
- район / место;
- тип объекта;
- количество листов;
- количество объявлений на А4;
- общий примерный объём объявлений;
- телефон для отклика;
- включённые настройки: QR, фото, линии реза, безопасные поля.

## Шаг 8. Отчёт

Шаг связан с блоком:

```text
#spnDistributionReport
```

Показывать:

- заполнение отчёта из задания;
- дату;
- место расклейки;
- количество расклеенных листов;
- звонки / сообщения;
- целевые обращения;
- следующий шаг;
- заметки;
- метрики;
- вывод по результату;
- сохранение в историю;
- повтор прошлой расклейки;
- CSV-экспорт.

Следующее улучшение: превратить историю отчётов в аналитику связок:

```text
шаблон + район + формат А4 + отклики
```

## Режимы

### Новичок СПН

Показывать только безопасные шаблоны и основные поля.

Цель: новый СПН должен сделать первый макет 2 на А4 без объяснений.

### Быстро

Основной стабильный режим для большинства пользователей.

Цель: не мешать опытному СПН, но не перегружать его всем набором дополнительных настроек сразу.

### Расширенно

Показывать все настройки, сохранение, отчёты и дополнительные инструменты.

Цель: дать гибкость менеджеру, опытному СПН или человеку, который готовит нестандартный макет.

4. Добавлены кнопки `Назад` и `Далее`.
5. Добавлен выбор количества объявлений на А4 в начале интерфейса.
6. Вариант `2 на А4` отмечен как частый.
7. Добавлены шаги `Задание` и `Отчёт`.
8. Шаги `Задание` и `Отчёт` закреплены в `validate:newbie-mode-docs`.
9. Категории заготовок оформлены как карточки решений.
10. Карточки шаблонов используют `office`-метаданные, если они есть.

## Что нужно улучшить дальше

1. В режиме `Новичок` включать пошаговый режим по умолчанию.
2. Разделить визуально цель и выбор количества на А4.
3. Добавить подсказку к каждому шагу.
4. Сделать кнопку `Далее` умнее: не блокировать агрессивно, а объяснять, что желательно заполнить.
5. Добавить менеджерский шаг или notice для шаблонов с тегом `менеджер`.
6. Расширить `office`-метаданные на локальные шаблоны Борисоглебска и ЖК «Теллерманов сад».
7. Добавить нормальные фото-компоновки для объектных и брендовых макетов.

## Критерии готовности

1. Новый СПН может сделать макет 2 на А4 без объяснений.
2. Формат листа выбирается до шаблона.
3. Пустые макеты не показываются новичку по умолчанию.
4. Сложные настройки скрыты, но доступны.
5. Макеты с фото имеют нормальную компоновку изображения.
6. Перед печатью есть понятная проверка.
7. После печати можно сформировать задание на расклейку.
8. После расклейки можно сохранить отчёт, повторить прошлую расклейку и экспортировать CSV.

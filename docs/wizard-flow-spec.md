# Wizard Flow v1

## Цель

Упростить создание макета расклейки. Пользователь должен идти по шагам, а не видеть все настройки сразу.

## Продуктовый принцип

```text
Новичок видит следующий нужный шаг.
Опытный СПН может раскрыть дополнительные настройки.
Менеджер видит проверку, риски и результат.
```

## Целевой порядок работы

```text
1. Цель
2. Количество объявлений на А4
3. Категория заготовок
4. Выбор макета
5. Редактирование текста
6. Фото и QR
7. Проверка
8. Печать / PDF
9. Задание на расклейку
10. Отчёт по результату
```

## Текущая реализация в коде

Сейчас `assets/js/spnWizardFlow.js` реализует 8 экранных шагов. Некоторые целевые пункты объединены, чтобы не переписывать ядро приложения резко.

```text
1. Цель       → цель + рабочая ситуация + выбор количества на А4
2. Заготовка  → категории заготовок + список шаблонов
3. Текст      → контакты, объект, заголовок, описание, преимущества
4. Фото / QR  → фото, режим фото, ссылка и подпись QR
5. Проверка   → настройки печати + контроль качества + сводка менеджера
6. Сохранить  → сохранение текущего и именованного макета
7. Задание    → задание на расклейку
8. Отчёт      → отчёт после расклейки, история, повтор и CSV
```

Каждый шаг показывает блок `Что сделать сейчас`. Подсказка меняется при переходе между шагами и объясняет ближайшее действие без блокировки интерфейса.

Кнопка `Далее` показывает название следующего шага, например `Далее: Заготовка`. Переход не блокируется жёстко. Если на текущем шаге пропущены желательные данные, мастер переходит дальше и показывает notice:

```text
Переход выполнен. Желательно вернуться и проверить
```

Такой формат помогает новичку, но не мешает опытному СПН продолжать работу.

Такой формат безопасен для текущей архитектуры: мастер управляет видимостью крупных секций, но не ломает `app.js` и не меняет хранение состояния.

## Шаг 1. Цель и количество на А4

Показывать крупные карточки целей:

- найти собственника;
- найти покупателя;
- продать объект;
- новостройка;
- оценка / услуга;
- аренда;
- личный бренд;
- частное объявление.

Рядом или сразу под целью показывать количество объявлений на А4:

```text
1 крупно
2 на А4 — чаще всего
4 экономно
6 мелко
8 мини
```

По умолчанию должен быть выбран вариант `2 на А4`.

Подсказки:

- `1 крупно` — объект, фото, витрина, объявление на стойку;
- `2 на А4` — основной рабочий формат для офиса;
- `4 экономно` — подъезды, короткие тексты, отрывные телефоны;
- `6 мелко` и `8 мини` — только очень короткий текст без перегруза.

Активная подсказка шага должна объяснять: для офиса чаще всего подходит 2 на А4, для подъездов — 4 на А4, для объекта с фото — 1 на А4.

При переходе дальше мастер мягко напоминает выбрать количество объявлений на А4, если формат не выбран.

## Шаг 2. Заготовка

Этот шаг объединяет выбор категории и выбор конкретного шаблона.

Категории уже оформлены как карточки решений. Пользователь выбирает рабочий сценарий, а не технический поисковый фильтр.

Категории:

- рекомендовано;
- новичку;
- менеджер;
- подъезд;
- собственники;
- покупатели;
- объекты;
- новостройки;
- Теллерманов сад;
- доверие;
- с фото;
- пустые под проверку менеджера.

Карточки шаблонов показывают бейджи и пояснения. Если у шаблона есть `office`-метаданные, интерфейс использует:

```text
office.level
office.risk
office.recommendedPrintCount
office.managerNote
```

Это позволяет объяснять, почему шаблон подходит новичку, почему нужен менеджер, какой риск у макета и какой формат А4 лучше использовать.

Если выбран шаблон уровня `manager` или с высоким риском, отдельный notice подсказывает: перед печатью покажите макет менеджеру.

Активная подсказка шага должна напоминать: новичку лучше брать `Рекомендовано` или `Новичку`, а шаблоны с пометкой `Проверка` нужно показать менеджеру.

При переходе дальше мастер мягко напоминает выбрать конкретный шаблон, если карточка макета не выбрана.

## Шаг 3. Редактирование текста

Каждый видимый текст в макете должен быть редактируемым.

Основные группы полей:

- контакты;
- объект / район;
- текст объявления;
- дополнительный блок;
- состав макета;
- порядок блоков.

В режиме `Новичок` нужно постепенно скрывать сложные настройки и показывать только то, что обязательно для выбранного шаблона.

Активная подсказка шага должна напоминать: текст должен быть коротким, а смысл объявления должен считываться за 2–3 секунды.

При переходе дальше мастер мягко проверяет телефон, заголовок и основной текст.

## Шаг 4. Фото и QR

Показывать фото и QR только когда они нужны выбранному макету или пользователь сам включил эти блоки.

Для фото нужны отдельные компоновки:

- крупное фото сверху;
- фото слева;
- карточка объекта с фото;
- планировка;
- изображение ЖК;
- фото СПН для личного бренда.

Фото не должно выглядеть узкой полоской.

Активная подсказка шага должна объяснять: для подъездной расклейки важнее крупный телефон, а для объекта, витрины или ЖК важно качество изображения и читаемость QR.

Если заполнена ссылка QR, но нет подписи, мастер мягко напоминает добавить понятную подпись.

## Шаг 5. Проверка и печать

Текущая реализация объединяет печатные настройки, контроль качества и менеджерскую сводку в один шаг.

На шаге должны быть видны:

- количество объявлений на листе;
- деление листа;
- линии реза;
- безопасные поля;
- отрывные телефоны;
- кнопка `Проверить`;
- список ошибок, важных замечаний и советов;
- сводка менеджера по выбранному шаблону;
- уровень шаблона из `office.level`;
- риск из `office.risk`;
- рекомендованный формат из `office.recommendedPrintCount`;
- заметка из `office.managerNote`;
- кнопка `Печать / PDF`.

Сводка менеджера должна помогать принять решение:

```text
Можно печатать.
Нужно исправить телефон.
Лучше показать менеджеру.
Макет перегружен — выберите 2 на А4 или сократите текст.
```

Если у шаблона нет `office`-метаданных, сводка должна честно показать, что структурированная офисная разметка пока не задана.

Активная подсказка шага должна напоминать: сначала нажать `Проверить`, затем исправить ошибки качества, проверить безопасные поля и только после этого печатать.

При переходе дальше мастер мягко проверяет качество не ниже 70 баллов, линии реза и безопасные поля. Это не заменяет строгую защиту печати в режиме `Новичок`.

## Шаг 6. Сохранить

Показывать:

- название макета;
- сохранение именованного макета;
- загрузку сохранённого макета;
- удаление сохранённого макета;
- сохранение последнего макета;
- открытие и скачивание JSON.

В режиме `Новичок` этот шаг не должен отвлекать до печати. Его можно показывать после проверки или после печати.

Активная подсказка шага должна предлагать сохранять удачные макеты с понятным названием: район, цель, формат А4.

## Шаг 7. Задание на расклейку

Шаг связан с блоком:

```text
#spnDistributionTask
```

Показывать:

- количество листов;
- место расклейки;
- ответственного;
- срок;
- текст задания;
- кнопку копирования.

Задание должно автоматически учитывать:

- заголовок макета;
- район / место;
- тип объекта;
- количество листов;
- количество объявлений на А4;
- общий примерный объём объявлений;
- телефон для отклика;
- включённые настройки: QR, фото, линии реза, безопасные поля.

Активная подсказка шага должна объяснять: сформируйте место, количество листов, ответственного и срок, затем скопируйте текст задания исполнителю.

При переходе к отчёту мастер мягко напоминает указать место расклейки и количество листов.

## Шаг 8. Отчёт

Шаг связан с блоком:

```text
#spnDistributionReport
```

Показывать:

- заполнение отчёта из задания;
- дату;
- место расклейки;
- количество расклеенных листов;
- звонки / сообщения;
- целевые обращения;
- следующий шаг;
- заметки;
- метрики;
- вывод по результату;
- сохранение в историю;
- фильтр истории;
- аналитику связок;
- повтор прошлой расклейки;
- CSV-экспорт.

История отчётов уже показывает:

```text
шаблон + район + формат А4 + отклики + целевые обращения
```

Активная подсказка шага должна напоминать: после расклейки внесите листы, звонки, целевые обращения и заметки, затем сохраните отчёт в историю.

## Режимы

### Новичок СПН

Показывать только безопасные шаблоны и основные поля.

Цель: новый СПН должен сделать первый макет 2 на А4 без объяснений.

### Быстро

Основной стабильный режим для большинства пользователей.

Цель: не мешать опытному СПН, но не перегружать его всем набором дополнительных настроек сразу.

### Расширенно

Показывать все настройки, сохранение, отчёты и дополнительные инструменты.

Цель: дать гибкость менеджеру, опытному СПН или человеку, который готовит нестандартный макет.

## Что нужно улучшить дальше

1. В режиме `Новичок` включать пошаговый режим по умолчанию.
2. Разделить визуально цель и выбор количества на А4.
3. Добавить нормальные фото-компоновки для объектных и брендовых макетов.

## Критерии готовности

1. Новый СПН может сделать макет 2 на А4 без объяснений.
2. Формат листа выбирается до шаблона.
3. Пустые макеты не показываются новичку по умолчанию.
4. Сложные настройки скрыты, но доступны.
5. Макеты с фото имеют нормальную компоновку изображения.
6. Перед печатью есть понятная проверка.
7. После печати можно сформировать задание на расклейку.
8. После расклейки можно сохранить отчёт, повторить прошлую расклейку и экспортировать CSV.

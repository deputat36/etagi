# Компактный статус пошаговой подготовки

Документ описывает модуль `assets/js/spnWizardStepStatus.js`.

## Назначение

Пошаговая подготовка содержит восемь шагов. Раньше активный шаг был виден, но для оценки общей готовности приходилось открывать каждый раздел отдельно.

Модуль добавляет:

- короткую метку на каждую кнопку шага;
- итог `До печати`;
- итог `После печати`;
- пояснение состояния в `title` метки.

Модуль не блокирует переходы и не запускает печать.

## Состояния

```text
готово       обязательные данные шага заполнены
проверить    есть пропуск или недостаточная проверка
необязательно фото, QR или сохранение сейчас не требуются
после печати шаг выполняется после подготовки листа
```

## Что считается готовым

### 1. Задача и формат

- выбрана задача;
- выбран формат А4.

### 2. Шаблон

- выбран конкретный шаблон;
- выбранный шаблон не помечен как устаревший.

### 3. Текст

- телефон содержит не меньше 10 цифр;
- заполнен заголовок;
- заполнено описание.

### 4. Фото / QR

Шаг необязателен, пока фото и QR выключены.

Если соответствующий блок включён:

- для фото выбран файл;
- для QR указана ссылка;
- для заполненной QR-ссылки добавлена подпись.

### 5. Проверка и печать

- качество не ниже 70;
- включены линии реза;
- включены безопасные поля.

### 6. Сохранение

Шаг не блокирует печать. Метка меняется на `есть макет`, если макет назван или уже есть именованные сохранения.

### 7. Задание

- указано место расклейки;
- количество листов больше нуля.

### 8. Отчёт

- указаны дата, место и количество листов текущего отчёта.

Если текущий отчёт ещё не заполнен, метка может показать количество записей в существующей истории.

## Общий итог

`До печати` считает обязательные шаги:

```text
Задача и формат
Шаблон
Текст
Проверка и печать
Фото / QR — только когда этот блок включён
```

`После печати` считает:

```text
Задание
Отчёт
```

Сохранение макета не входит в обязательный итог.

## Безопасность runtime

Модуль:

- подключается через `spnUiMode.js`;
- ждёт существующий `spnWizardFlow` ограниченное число попыток;
- использует `requestAnimationFrame` для объединения обновлений;
- слушает `input`, `change` и ограниченный `click` только внутри `.sidebar`;
- не использует глобальный `document.addEventListener('click', ...)`;
- не записывает новые ключи в `localStorage`;
- не использует `setInterval`;
- не изменяет поля формы и не вызывает `.click()` на рабочих кнопках.

## Ручная проверка

1. Открыть генератор.
2. Проверить наличие меток у восьми шагов.
3. Убедиться, что итог показывает `До печати` и `После печати`.
4. Выбрать цель и формат — шаг 1 должен стать готовым.
5. Выбрать обычный рабочий шаблон — шаг 2 должен стать готовым.
6. Выбрать устаревший шаблон — шаг 2 должен показать `заменить`.
7. Очистить телефон — шаг 3 должен показать неполную готовность.
8. Заполнить телефон, заголовок и описание — шаг 3 должен стать готовым.
9. Выключить фото и QR — шаг 4 должен показать `необязательно`.
10. Включить QR без ссылки — шаг 4 должен показать `проверить`.
11. Запустить контроль качества, включить линии реза и безопасные поля — шаг 5 должен стать готовым при результате от 70.
12. Заполнить место и листы задания — шаг 7 должен стать готовым.
13. Заполнить дату, место и листы отчёта — шаг 8 должен показать `заполнен`.
14. Переключить режимы `Новичок`, `Быстро`, `Расширенно` и убедиться, что метки не мешают навигации.
15. Проверить мобильную ширину: метки не перекрывают название шага.
16. Открыть печать или PDF: служебные метки не должны попадать на лист А4.

## Автоматическая проверка

```bash
npm run validate:wizard-step-status
npm run smoke:browser
```

Browser smoke фактически проверяет восемь бейджей и оба общих итога, переводит выключенные фото и QR в состояние `необязательно`, заполняет задание и отчёт, а затем проверяет готовность этапа `После печати`. После загрузки тестовых изображений статус медиа должен вернуться в `готово`.

Исходный аудит выполнен на `main`:

```text
bb24feb2d71f11a2c0f8ee35379530b5c418200e
```

Область:

```text
рабочая ситуация → задача → сценарий → шаблон
```

Machine-readable карта:

```text
data/workflow-selection-audit.json
```

## Новое поведение

### Рабочая ситуация

Владелец интерфейса: `assets/js/spnWizard.js`.

Владелец изменения рабочего состояния: `assets/js/app.js`.

Выбор ситуации отправляет одно событие `spn:workflow-selection`. Обработчик за одну транзакцию:

- устанавливает задачу `goal`;
- подставляет поисковую фразу;
- сбрасывает плотность поиска в `Все`;
- применяет рекомендованные формат А4 и компоновку;
- сбрасывает сценарий в `Все`;
- снимает `templateId`;
- выполняет один `renderAll()`.

Программный клик по задаче и задержка 90 мс удалены.

Текущий макет не заменяется до явного выбора шаблона. Пользователь получает прямое сообщение, что текущий текст сохранён и новый шаблон нужно выбрать карточкой.

### Задача

Выбор задачи теперь:

- меняет `state.goal`;
- сбрасывает сценарий в `Все`;
- снимает связь с ранее выбранной карточкой через `state.templateId = ''`;
- фильтрует библиотеку;
- сохраняет текущий заголовок, описание, преимущества, название макета, профиль и фотографии.

Первый подходящий шаблон автоматически не применяется.

### Сценарий

Сценарий остаётся дополнительным фильтром и не меняет макет.

### Шаблон

Только явный клик по карточке вызывает `applyTemplate()` и заменяет текст и исходную компоновку. После клика активная карточка соответствует фактически применённому `templateId`.

## Исправленные findings

### WF-01 — исправлено

Ранее кнопка задачи автоматически применяла первый найденный шаблон. Теперь задача только фильтрует библиотеку, а текст не меняется до явного выбора.

### WF-02 — исправлено

Ранее ситуация сначала программно нажимала задачу, а затем через таймер подставляла поиск и настройки. Теперь вся операция передаётся одним событием и завершается одним рендером.

### WF-06 — исправлено

Ранее автоматически применённый шаблон мог исчезнуть после поисковой фильтрации, оставаясь скрытым источником текста. Теперь до явного клика `templateId` пуст, поэтому скрытого активного шаблона нет.

### WF-04 — исправлено

Ручной поиск теперь только фильтрует библиотеку. Шаг «Ситуация» закрывается исключительно после явного выбора офисной ситуации, поэтому поиск по слову «квартира» больше не создаёт ложный прогресс маршрута.

## Оставшиеся вопросы

WF-03 и WF-05 остаются открытыми:

- WF-03 — поиск ситуации и сценарный фильтр частично дублируют друг друга;
- WF-05 — явный шаблон синхронизирует задачу, но активный сценарий отдельно не пересчитывается.

Следующая безопасная итерация должна упростить повторяющиеся фильтры, а затем определить сценарий после явного выбора шаблона без изменения рекламных текстов и печатной матрицы.

## Проверка

Статический контракт:

```bash
node tools/validate-workflow-selection-audit.mjs
```

Browser smoke:

```bash
node tools/run-workflow-selection-smoke.mjs
```

Smoke подтверждает:

- ситуация выбирает задачу, поиск, формат и компоновку;
- название и текст текущего макета сохраняются;
- до явного клика активной карточки нет;
- сценарий только фильтрует;
- шаблон применяется явным кликом;
- ручной выбор задачи не заменяет макет;
- ручной поиск не засчитывается как офисная ситуация;
- failure-log содержит последнее действие и снимок состояния.

# Чек-лист проверки backup рабочего пространства

Проверять после изменений storage, localStorage-ключей, отчётов, Wizard Flow, менеджерской проверки или модуля backup.

## Подготовка

1. Открыть генератор в режиме `Расширенно`.
2. Сохранить профиль СПН.
3. Добавить один макет в избранное.
4. Сохранить именованный макет.
5. Изменить режим интерфейса.
6. Сохранить хотя бы один отчёт после расклейки.
7. Запомнить значения телефона, названия макета и количество отчётов.

## Экспорт

1. Нажать `Скачать backup`.
2. Проверить имя файла `etagi-workspace-backup-YYYY-MM-DD.json`.
3. Открыть файл как текст.
4. Проверить поля:
   - `schema`;
   - `version`;
   - `exportedAt`;
   - `appVersion`;
   - `entryCount`;
   - `entries`.
5. Убедиться, что все ключи внутри `entries` начинаются с `etagi-raskleyka-`.
6. Убедиться, что в файле нет base64-фотографий из полей `photoOne` и `photoTwo`.

## Импорт с объединением

1. Изменить профиль СПН после экспорта.
2. Создать ещё один именованный макет.
3. Выбрать `Объединить с текущими данными`.
4. Открыть сохранённый backup.
5. Проверить текст подтверждения с количеством разделов.
6. Отменить подтверждение и убедиться, что данные не изменились.
7. Повторить импорт и подтвердить.
8. После перезагрузки проверить восстановленный профиль и историю.
9. Проверить, что ключи, отсутствующие в backup, не были удалены.

## Импорт с заменой

1. Снова создать свежий backup.
2. Изменить профиль, удалить именованный макет и добавить новый отчёт.
3. Выбрать `Заменить рабочее пространство`.
4. Открыть backup.
5. Проверить отдельное предупреждение об удалении текущих разделов.
6. Подтвердить.
7. После перезагрузки убедиться, что состояние соответствует backup, а более новые локальные изменения исчезли.

## Неверные файлы

Проверить, что текущие данные не меняются при попытке открыть:

1. файл, который не является JSON;
2. JSON с другой схемой;
3. JSON с неизвестной версией;
4. JSON без `entries`;
5. JSON с ключом без префикса `etagi-raskleyka-`;
6. JSON с нестроковым значением;
7. пустой backup;
8. файл больше 3 МБ.

## Откат при ошибке

Сценарий удобнее проверять в DevTools или отдельной тестовой среде с искусственно ограниченной квотой localStorage.

1. Сохранить текущее рабочее пространство.
2. Запустить импорт в режиме `Заменить`.
3. Вызвать ошибку записи localStorage.
4. Проверить сообщение `Предыдущие данные возвращены без изменений`.
5. Перезагрузить страницу.
6. Убедиться, что профиль, макеты, избранное и отчёты остались в состоянии до импорта.

## Приватность

1. Убедиться, что импорт и экспорт не выполняют сетевые запросы.
2. Убедиться, что используется только `localStorage`, `Blob` и локальный выбор файла.
3. Убедиться, что backup не публикуется и не отправляется на внешний сервер.

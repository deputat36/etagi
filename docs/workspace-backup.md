# Backup рабочего пространства

Полный backup предназначен для переноса генератора на другой компьютер, восстановления после очистки браузера и сохранения рабочих данных перед крупным обновлением.

## Что входит в backup

Модуль экспортирует все записи `localStorage`, ключ которых начинается с:

```text
etagi-raskleyka-
```

Поэтому в файл попадают:

- профиль СПН;
- текущий автосохранённый макет;
- последний сохранённый макет;
- именованные макеты;
- избранные шаблоны;
- режим интерфейса;
- состояние Wizard Flow;
- задания на расклейку;
- менеджерские проверки;
- история отчётов и аналитики;
- другие совместимые настройки проекта.

Фото не входят в backup. Генератор намеренно не хранит изображения в `localStorage`, чтобы не переполнять память браузера.

## Как скачать backup

1. Открыть режим `Расширенно`.
2. Найти блок `Сохранение`.
3. В разделе `Backup рабочего пространства` нажать `Скачать backup`.
4. Сохранить JSON-файл в надёжное место.

Имя файла имеет формат:

```text
etagi-workspace-backup-YYYY-MM-DD.json
```

## Как восстановить данные

1. Нажать `Открыть backup`.
2. Выбрать JSON-файл.
3. Выбрать режим восстановления.
4. Прочитать подтверждение с количеством текущих и импортируемых разделов.
5. Подтвердить операцию.
6. Дождаться автоматической перезагрузки страницы.

## Режимы импорта

### Объединить с текущими данными

Backup добавляется к текущему рабочему пространству. Совпадающие ключи обновляются значениями из файла. Остальные текущие ключи сохраняются.

Этот режим подходит для обычного переноса и восстановления отдельных недостающих данных.

### Заменить рабочее пространство

Все ключи проекта с префиксом `etagi-raskleyka-` удаляются и заменяются содержимым backup.

Этот режим подходит для полного восстановления точного состояния другого компьютера или предыдущей рабочей копии.

Перед заменой требуется отдельное подтверждение.

## Защита данных

Модуль проверяет:

- схему `etagi-raskleyka-workspace-backup`;
- версию формата;
- максимальный размер 3 МБ;
- наличие объекта `entries`;
- префикс каждого ключа;
- строковый формат каждого значения localStorage;
- наличие хотя бы одного раздела проекта.

Файл не может записывать произвольные ключи другого сайта или приложения.

Перед импортом создаётся снимок текущего рабочего пространства. Если браузеру не хватает места или запись завершается ошибкой, модуль удаляет частичный результат и возвращает предыдущие данные.

## Ограничения

- backup хранит только локальные данные этого браузера;
- фото нужно переносить отдельно;
- файл может содержать телефоны, имена, адреса и заметки, поэтому его нельзя публиковать в открытом доступе;
- после восстановления нужно проверить профиль, сохранённые макеты и несколько последних отчётов;
- неизвестная версия backup отклоняется без изменения текущих данных.

## Рекомендуемый порядок

Создавать backup:

- перед очисткой браузера;
- перед сменой компьютера;
- перед крупным обновлением генератора;
- не реже одного раза в месяц при активном использовании истории отчётов.
